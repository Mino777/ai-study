# 🔁 복습 데크 (Spaced Repetition) — 항상 되돌아오는 곳

> **왜:** 박제만 하면 안 읽는다. 능동인출(active recall) + 간격반복(SRS)으로 배운 걸 장기기억에 박는다.
> **Claude 운영 규약:** **매 세션 시작 시** 이 데크에서 `다음복습 ≤ 오늘`인 카드를 골라 **1~2개 능동인출 퀴즈**로 던진다(답을 먼저 물어보고, 그다음 카드 링크로 확인). 유저가 맞히면 박스 +1(간격 늘림), 틀리면 박스 1로 리셋. 결과에 따라 `다음복습`·`박스` 갱신 후 커밋. 세션에서 **새 개념을 배우면** 이 데크에 카드 1줄 추가(박스 1, 다음복습=내일).

## 📦 Leitner 박스 간격
```
박스1: +1일   박스2: +3일   박스3: +7일   박스4: +16일   박스5: +35일(정착)
정답 → 박스 +1 (간격 늘림)   ·   오답 → 박스 1로 리셋
```

## 📝 정답 키 (인출 실패 시 확인용)
- **5-스텝:** ① 번역(값/인덱스) ② brute force ③ 낭비 찾기 ④ 도구 매칭 ⑤ 복잡도+엣지
- **해시 심장:** "미래의 짝을 찾지 말고, 과거에 본 것을 기억했다 되물어라" (해시 조회 O(1))
- **4형제:** Two Sum=딕셔너리[값→인덱스] · Contains Dup=Set · Valid Anagram=딕셔너리[글자→개수] · Group Anagrams=딕셔너리[지문→리스트]
- **COW:** Copy-on-Write. 축2개 = [대입 복사비용] vs [힙·ARC·동적디스패치 오버헤드]
- **투포인터:** "정렬"·"양끝/회문" → 투 포인터. Valid Palindrome은 **공간**을 O(n)→O(1)로 (시간 아님). Two Sum II는 정렬 활용해 해시 불필요
- **아키텍처 서사틀:** ①상황 ②결정 ③트레이드오프 ④결과. 결정타 = 비용만 말하지 말고 **"어떻게 줄였나(mitigation)"**까지
- **모바일 SD 규모:** 서버 QPS 아님 → **사용자당 자원 예산**(캐시·메모리·데이터·배터리)
- **트레이드오프:** "메모리를 대가로 **시간(속도)**를 산다". 해시·캐시·메모이제이션·DP
- **Big-O 루틴:** ①중첩 몇겹→지수 ②정렬/반씩→log ③숨은 O(n)(contains/sort)→곱 ④큰 항만. for 안 contains = 숨은 O(n²)
- **클라-서버:** 클라=요청하는 손님(UI) / 서버=진실 소유한 주방(데이터·로직). 가격을 클라에서 계산하면 조작 가능 → 서버가 진실의 원천
