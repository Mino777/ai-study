# ⏱️ 시간복잡도 3초 계산법 (Big-O 눈대중 치트시트)

> W1 · 2026-07-23 · 목표: 문제를 보면 3초 안에 Big-O 각을 잡는다. 눈으로 패턴만 매칭.

## 🎯 핵심 한 문장
> **"가장 안쪽 핵심 연산이, n이 커질 때 총 몇 번 실행되나?"** — 그것만 세면 끝. 나머진 이 질문의 변형.

---

## 꿀팁 1 — 반복문 **중첩 깊이 = n의 지수**
```
반복문 없음     →  O(1)     n 상관없이 고정
for 한 개       →  O(n)     전체 1번 훑음
for 안에 for    →  O(n²)    각 원소마다 전체 훑음 (중첩 2단)
for·for·for     →  O(n³)    중첩 3단
```
👉 **중첩 for 개수를 세라.** 2겹이면 대체로 n², 3겹이면 n³.

## 꿀팁 2 — "절반씩 줄어들면" = log n
```
n → n/2 → n/4 → ... → 1     반씩 버리면 log₂n 번
이진 탐색(정렬 데이터 반씩)   →  O(log n)
정렬(merge/quick)          →  O(n log n)   "훑으면서 반씩"
힙 push/pop                →  O(log n)
```
👉 "매 단계 문제가 반으로 준다" 싶으면 log. **정렬이 끼면** 거의 항상 `n log n`.

## 꿀팁 3 — 코드 구조 → Big-O 즉시 매칭
```
┌─────────────────────────────┬──────────────┐
│ 코드에서 이게 보이면          │ Big-O        │
├─────────────────────────────┼──────────────┤
│ 인덱스 접근 arr[i], 산술      │ O(1)         │
│ 딕셔너리/Set 조회·삽입        │ O(1) 평균     │
│ for 한 바퀴                  │ O(n)         │
│ 중첩 for 두 바퀴             │ O(n²)        │
│ .sorted() / 정렬            │ O(n log n)   │
│ 이진 탐색 / 반씩             │ O(log n)     │
│ 모든 부분집합 (백트래킹)      │ O(2ⁿ)        │
│ 모든 순열                    │ O(n!)        │
└─────────────────────────────┴──────────────┘
```

## ⚠️ 함정 3개 (여기서 다들 틀림)

**함정 1 — 상수·낮은 차수는 버린다**
```
O(2n + 3)  →  O(n)      계수·상수 버림
O(n² + n)  →  O(n²)     지배 항목(큰 놈)만 남김
```

**함정 2 — 따로 있는 loop는 곱이 아니라 합**
```
for {}  for {}          나란히 = O(n)+O(n) = O(n)
for { for {} }          중첩   = O(n)×O(n) = O(n²)
```
👉 중첩(안에 들어감)이면 곱, 나란히면 합.

**함정 3 — 숨은 비용 (제일 자주 놓침)**
```
array.contains(x)   // O(n)! 배열 전체 훑음
string + string     // O(n)
array.sorted()      // O(n log n)
```
👉 for 안에서 `contains()` 쓰면 O(n)×O(n) = **숨은 O(n²)**. → 그래서 Set/Dict로 O(1) 치환.

---

## 🎤 면접용 3초 루틴
```
1. 중첩 for 몇 겹?          → n의 지수
2. 정렬/반씩 있나?          → log 곱하기
3. 안쪽에 숨은 O(n)? (contains/sort/slice) → 곱해주기
4. 제일 큰 항만 남기고 계수 버리기
```

## 🔬 오늘의 해시 4형제로 검증
```
Two Sum:        for × 딕셔너리조회 O(1)        =  O(n)
Contains Dup:   for × Set조회 O(1)            =  O(n)
Valid Anagram:  for × 딕셔너리 O(1)            =  O(n)
Group Anagrams: for n × 각 단어 정렬 O(k log k) =  O(n·k log k)
                └ 단일 loop이어도 안쪽 정렬 비용을 곱함 (숨은 비용!)
```

**한 줄 요약:** *중첩 세고 → 정렬/반씩이면 log 붙이고 → 숨은 contains/sort 곱하고 → 큰 항만 남긴다.*
