# W1-01 · 배열 & 해시맵, 그리고 Big-O (시각적 레슨)

> 시각적 교육 스타일의 첫 완성 레슨. 모든 개념을 **그림 먼저 → 코드 나중**. 내일 세션 1의 기반.

---

## 1. Big-O를 "느낌"으로 — 성장 곡선

알고리즘이 느린지 빠른지는 **입력이 커질 때 연산이 얼마나 폭발하는가**로 잰다.

```
연산 횟수
  ▲
  │                                      ╱  O(n²)  모든 짝 비교
  │                                  ╱        → n 늘면 제곱으로 폭발 😱
  │                              ╱
  │                          ╱
  │                      ╱
  │                  ╱
  │              ╱ ┈┈┈┈┈┈┈┈┈┈┈┈┈┈┈  O(n)  한 번 훑기
  │          ╱ ┈┈┈┈┈┈┈
  │      ╱┈┈┈┈
  │  ╱┈┈  ─────────────────────────  O(1)  즉시 (최고)
  └──────────────────────────────────▶  입력 크기 n
```

| 표기 | 별명 | n=10 | n=1,000 | 체감 |
|------|------|------|---------|------|
| `O(1)` | 상수 | 1 | 1 | 순간이동 |
| `O(n)` | 선형 | 10 | 1,000 | 걷기 |
| `O(n²)` | 제곱 | 100 | 1,000,000 | 🐢 |

> 코테의 절반은 **"O(n²)를 O(n)으로 줄이기"** 게임이다. 그 무기가 바로 해시맵.

## 2. 배열 검색 vs 해시맵 검색 — 왜 해시맵이 빠른가

**배열에서 "7 어딨어?" → 하나씩 다 봐야 함 = O(n)**
```
   찾는 값: 7
        ┌────┬────┬────┬────┐
        │ 2  │ 7  │ 11 │ 15 │
        └────┴────┴────┴────┘
          ✗    ✓                 ← 운 나쁘면 끝까지 (n번)
```

**해시맵은 "계산 한 번"으로 바로 그 칸 = O(1)**
```
   찾는 값: 7
        해시함수(7) ──▶ 📦 "7번 서랍" 즉시 열기
        (몇 번째인지 세지 않고 계산으로 바로 점프)
```

> 비유: 배열은 **"1번 사물함부터 다 열어보기"**, 해시맵은 **"이름표 보고 그 사물함으로 직행"**.

## 3. Two Sum — 메모리 추적 (핵심 패턴)

**문제:** 더해서 `target`이 되는 두 칸의 **번호** 찾기.
**아이디어:** *"짝꿍을 미리 메모해두면 다시 안 찾아도 된다."*

```
배열: [3, 2, 4]     target = 6
메모지(seen): { }    ← "본 값 → 그 칸 번호" 저장

┌─ 칸0 ─ 값=3 ────────────────────────────────┐
│ 짝꿍 = 6 - 3 = 3    "3 메모지에 있나?" { } ❌  │
│ → 메모지에 기록      seen = { 3:0 }           │
└──────────────────────────────────────────────┘
┌─ 칸1 ─ 값=2 ────────────────────────────────┐
│ 짝꿍 = 6 - 2 = 4    "4 있나?" {3:0} ❌         │
│ → 메모지에 기록      seen = { 3:0, 2:1 }      │
└──────────────────────────────────────────────┘
┌─ 칸2 ─ 값=4 ────────────────────────────────┐
│ 짝꿍 = 6 - 4 = 2    "2 있나?" {3:0,2:1} ✅ 1번칸! │
│ 🎉 정답 [1, 2]      (2 + 4 = 6)               │
└──────────────────────────────────────────────┘
```

배열을 **딱 한 번** 훑었다 → **O(n)**. (짝을 다 비교하는 브루트포스는 O(n²))

### 코드 (그림을 그대로 옮긴 것뿐)
```swift
func twoSum(_ nums: [Int], _ target: Int) -> [Int] {
    var seen: [Int: Int] = [:]          // 메모지: 값 → 칸번호
    for (i, n) in nums.enumerated() {    // 칸을 하나씩
        let mate = target - n            // 내 짝꿍은?
        if let j = seen[mate] {          // 짝꿍이 메모지에 있나?
            return [j, i]                // 있으면 정답!
        }
        seen[n] = i                      // 없으면 나를 메모
    }
    return []
}
```

> 당신이 떠올린 `enumerated()` 감각이 정확히 여기 쓰인다. 방향은 맞았어요.

---

## 🎯 이 레슨의 1문장 요약
**"다시 찾을 걸 메모지(해시맵)에 적어두면, 훑기 한 번(O(n))으로 끝난다."** — 이게 코테 1번 패턴.
