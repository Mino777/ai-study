# 🎨 시각적 교육 도구상자 (Claude 운영 규칙)

> 유저 요청: "개념·알고리즘 설명은 최대한 그림으로 시각적으로." **모든 레슨은 그림 먼저 → 코드 나중.** 텍스트 나열 금지.
> 완성 예시: `lessons/w1-01-arrays-hashing.md`.

## 🧭 문제 공략 5-스텝 (매 DSA 문제마다 명시적으로 호명 — 습관화 목적)

> 유저 요청(2026-07-23): "이 사고흐름을 앞으로도 반복적으로 계속 말해줘. 습관처럼 인지하게."
> → **모든 알고리즘 문제 풀이는 이 5스텝을 소리내어 밟는다.** 정답보다 이 사고 서사가 면접 점수. 라이브 플랫폼 그라인딩 대신 이 프레임을 머리로 돌리는 훈련이 평일 세션의 본체.

```
① 번역   문제를 그림으로. "주어진 것 / 원하는 것" 명확히 (값? 인덱스?)
② 무식하게 손으로 풀기   brute force로 일단 답을 낸다 (항상 됨. 느려도 됨)
③ 낭비 지점 찾기   "어디서 똑같은 일을 반복하지?" ← 핵심 통찰 지점
④ 낭비 제거 도구 매칭   그 반복을 없애는 자료구조/기법 (해시? 정렬? 포인터?)
⑤ 복잡도 + 엣지   O(?) 확인 + 빈 배열·중복·못 찾는 경우
```

면접 서사: "일단 brute force → 개선점 짚고 → 최적화" 흐름 자체가 시니어 감별 포인트.

## 상황별 시각화 형식 (골라 쓴다)

| 개념 유형 | 시각화 형식 | 예 |
|-----------|-------------|-----|
| 배열/인덱스 | 칸 그리드 `┌─┬─┐` + 인덱스 라벨 | Two Sum, 슬라이딩윈도우 |
| 포인터/참조/링크드리스트 | **box-and-pointer** (박스 + 화살표) | 값타입 vs 참조타입, LinkedList |
| 재귀/DFS | **콜스택 프레임** 쌓기/펼치기 | 트리 순회, 백트래킹 |
| BFS/그래프 | 노드 + 방문표시 + 큐 상태 스텝별 | 최단경로, 섬 개수 |
| 복잡도 | **성장 곡선 ASCII 차트** + n별 표 | Big-O |
| 알고리즘 진행 | **스텝별 상태 추적 박스** (변수 값 변화) | 해시맵 채워지는 과정 |
| 메모리/ARC | 힙/스택 박스 + retain count 숫자 | retain cycle, weak/strong |
| 동시성 | 타임라인 레인(스레드별 가로줄) | async/await suspend/resume |
| 아키텍처/시스템 | 계층 박스 + 데이터 흐름 화살표 (Mermaid) | MVVM, 모바일 SD |

## 원칙
1. **그림 먼저.** 개념을 그림으로 이해시킨 뒤에만 코드를 보여준다. 코드는 "그림을 옮긴 것"으로 제시.
2. **비유 하나씩.** 각 개념에 일상 비유 (해시맵=이름표 사물함, 인덱스=색인).
3. **1문장 요약.** 레슨 끝엔 굵은 한 줄로 핵심 압축.
4. **외부 시각 도구:** 자료구조 첫 노출은 [VisuAlgo](https://visualgo.net/en) 애니메이션 병행.
5. 위키 엔트리로 승격 시 Mermaid + 코드블록 조합 사용 (`content/` 규칙 준수).
