# 🎯 쿠팡 인터뷰 프로세스 리허설 (1순위 타겟)

> **가장 중요한 반전:** 쿠팡은 클래식 "iOS 면접"(ARC·RxSwift·GCD 트리비아)을 **안 본다**. **FAANG/아마존식 루프** — DSA 라이브코딩 + 시스템 디자인 + Leadership Principle 행동면접 — 을 CS 기초 기준으로, 종종 **영어**로, **Bar Raiser**까지 두고 본다.
> → **아마존 면접 보듯 준비한다.** 한국 SI식 iOS 지식 암기가 아니라. 당신의 UIKit/Rx/아키텍처 강점은 *이력서 방어*와 *시스템 디자인*에서 빛나지, 면접이 그걸 직접 시험하진 않는다.

---

## 1. 단계별 타임라인

```
 0.리크루터    1.온라인CT       2.폰스크린      3.온사이트 루프          4.디브리프+BarRaiser
 (30분,핏)  →  (HackerRank   →  (60분,        →  (경력L5: 4라운드    →  (히committee
               2~5문제)         라이브20~30분)     2코딩+1SD+1행동)        BarRaiser 합의)
                                                  KR~3h / 글로벌~4h        오퍼 ~1주
```

전체 ~25일, 난이도 3.2/5. **루프 크기는 팀/레벨별 2~4라운드로 유동** (경력직은 압축되어 2라운드=1h코딩+1h LP, SD 없는 경우도 보고됨). CT는 경력직에게 더 쉽고 구현 위주거나 생략되기도 — **하지만 있다고 가정하고 준비.**

## 2. 온라인 CT
- **플랫폼: HackerRank.** 2~5문제 / 60~150분.
- 난이도: **LeetCode medium 중심 + 일부 hard.** 배열/문자열/해시/스택·큐·힙/정렬/그리디/이분탐색/**DFS·BFS·그래프/DP/구간**.
- 경력직: 구현 위주로 쉬워지는 경향. **생략 기대 금지.**

## 3. 라이브 코딩
- **공유 에디터**(HackerRank 화면공유 또는 글로벌은 **CoderPad**), **페어 프로그래밍 방식**. 라운드당 **45~60분, ~1문제**. 온사이트 총 2~3문제.
- **평가:** Java/CS 기초, 깨끗한/버그 없는 코드, **엣지케이스 식별**, **시간·공간복잡도 분석**, 그리고 **커뮤니케이션**(특히 2번째 기술 라운드는 커뮤니케이션 중시). 조용히 코딩하면 탈락.
- **보고된 문제:** Min Stack(O(1)), Sort Colors, **LFU Cache(20분 내 기대)**, Best Meeting Point, 나선/대각 행렬 순회, 섬/그래프 거리, 부분수열 카운팅, 파일시스템 연산.

### 🗣️ Think-aloud 프로토콜 (매 라운드 무조건)
```
1. 문제 재진술 + 명확화 질문 (문제는 의도적으로 모호함!)
2. 브루트포스 + 복잡도 소리내어
3. 최적화 접근 + 목표 복잡도를 코딩 前 선언
4. 코딩하며 계속 내레이션 + 엣지케이스 선제 언급
5. 예시 + 코너케이스로 dry-run
6. 최종 시간/공간 복잡도 진술
```

## 4. 시스템 디자인 (모바일 — 당신에게 가장 큰 기회)
- 시니어 루프에 **전용 SD 라운드 1개.** 가상 공유툴(물리 화이트보드 아님).
- **iOS 지원자는 모바일 SD** (백엔드 분산시스템 아님). "Design a mobile app" 류 — 피드/이미지로더/채팅/다운로더 클라이언트 아키텍처 + 클라↔서버 데이터흐름 설명. **MVVM/Clean 아키텍처 논의 반복** → 당신 강점!
  > *신뢰도 주의: 모바일-vs-백엔드 구분은 Blind 직원 답변 1건 기반(방향성 강하나 얇음). 리크루터에게 SD 범위 확인. 백엔드 SD(이커머스/알림)는 스킴만.*

### 🏗️ 모바일 SD 답변 뼈대 (암기, ~45~60분)
```
1. 요구사항(5~10분)  기능(핵심3~5) vs 비기능(오프라인/실시간/대역폭/배터리) + 범위밖 명시 → 확인
2. API/인터페이스(5~10분)  REST/GraphQL/WebSocket/gRPC 트레이드오프. 페이지네이션 cursor>offset
3. 데이터 모델(5분)  로컬저장(Core Data/키밸류/보안저장) 선택
4. 클라 아키텍처(10~15분)  presentation/domain/data 계층 + MVVM + 네트워킹(재시도/토큰갱신) + repository/DI + 이미지로더
5. 네트워킹·캐싱  재시도/요청 dedup/캐시 전략
6. 오프라인·동기화  뮤테이션 큐 + client-generated ID(dedup) + 충돌해결 + sync 엔진
7. 성능  배터리/메모리/네트워크 가변성/크래시 방지
8. 보안  보안저장/인증/전송암호화/프라이버시
9. 딥다이브(15~20분)  면접관이 한 흐름 지목 → 상세화
10. 마무리(1~5분)  결정 요약 + 트레이드오프 (완벽한 답보다 트레이드오프가 중요)
```
**연습 문제:** 뉴스피드/트위터 클라, 인스타그램, 채팅(WhatsApp), 이미지 캐싱 라이브러리, 파일 다운로더(range로 일시정지/재개), 유튜브, 지도. (프레임워크명: SCADET)

## 5. 행동면접 — 쿠팡 15 Leadership Principles (아마존 near-clone)

**SBI(Situation→Behavior→Impact)** 권장(STAR 호환). **원칙당 2~3개 스토리 + 반드시 데이터/지표 포함.** 시니어는 디렉터가 진행하고 **비중 매우 큼 — LP 약하면 코딩 잘해도 침몰.**

1. Wow the Customer (고객집착)
2. Ruthless Prioritization *(쿠팡 고유)*
3. Company-wide Perspective (오너십)
4. Dive Deep
5. Think Systematically *(쿠팡 고유)*
6. Simplify
7. Disagree and Commit
8. Hire and Develop the Best
9. Deliver Results with Grit
10. Influence without Authority *(쿠팡 고유)*
11. Aim High and Find a Way (Think Big)
12. Learn Voraciously
13. Demand Excellence
14. Move with Urgency (Bias for Action)
15. Hate Waste (Frugality)

**단골:** 고객중심 결정 / 상사·동료와 이견(Disagree&Commit) / 하고싶던 걸 포기한 우선순위 결정 / 데이터 파고들어 문제해결(Dive Deep) / 마감 압박 속 blocker 돌파(Grit) / 권한없이 영향력 / jaw-dropping 결과.

## 6. Bar Raiser
- 쿠팡 공식 확인. 채용팀 외부의 **보정된 면접관**, 매 루프 참여, 15 LP·기술·장기잠재력을 객관 평가, 편향 상쇄. **아마존 동급 = 오퍼 거부권 있다고 간주.**

## 7. iOS 기술 깊이 = "프로젝트 딥다이브 방어"
- 전용 iOS 도메인 라운드 증거 없음. **가장 어려운 질문은 프로젝트 진위** — "진짜 네가 만들고 배포했나? 정확히 어떻게?" → **모든 프로젝트를 자기 코드처럼 방어.** 아키텍처/왜/바꾼다면/실제 배포 여부.
- 쿠팡 스택(방어용): Swift+ObjC, UIKit/Storyboard+SwiftUI, **MVVM 표준**, A/B, 3000+ 빌드/일.

## 8. 영어 & 글로벌 조직
- 글로벌 엔지니어링 조직(시애틀 HQ). **영어 면접은 실재하나 팀별로 상황적** — 미국/글로벌 조직 보고 팀일수록 영어. 통역 요청 가능.
- **리크루터에게 내 루프 언어 확인** + 자기소개·LP 스토리는 영어로 준비.

## 9. 🧘 당황 포인트 & 대응 (면접 당일 안 놀라기)

| 당황 트리거 | 진실 | 대응 |
|---|---|---|
| **문제가 일부러 모호함** (공식) | 모호함 대처력 테스트 | 침착히 명확화 질문 + 가정 소리내기 → **점수 획득** |
| 면접관이 중간에 반박 | 코칭 수용성/소통 테스트 | 피드백 수용, 얼지 말고 방어적이지 말 것 |
| 뜬금없는 질문 | 정상 | 심호흡 후 구조화 |
| 3~4h 마라톤, 매 라운드 새 면접관 | 표준 | 페이스 조절, 라운드 간 리셋 |
| **라운드 순서 랜덤** (행동이 먼저일 수도) | 흔함 | 코딩·행동 **동등하게** 준비 |
| **프로젝트 깊게 파고듦** | 가장 어려운 부분(보고됨) | 딥다이브 리허설, 내 코드 완벽히 숙지 |
| 조용한 코더는 2라운드 탈락 | 커뮤니케이션 가중 | 계속 think-aloud |

## 10. ✅ 쿠팡 맞춤 준비 체크리스트 (base 커리큘럼 위 델타)

**🔴 DSA (최대 갭·1순위 게이트):** LeetCode **medium 중심 100~150문제** + 일부 hard. LRU/**LFU 캐시 20분 내**. HackerRank/CoderPad에서 **영어로 소리내며** 복잡도 선언 연습. 누가 중간에 끊는 페어 시뮬레이션.
**🟠 모바일 SD:** §4 10단계 뼈대 암기(45분 타임박스) + 목 5회(뉴스피드/이미지캐시/채팅/오프라인동기화/다운로더). MVVM·계층 아키텍처 + **트레이드오프** 설명 = 강점 활용.
**🟡 LP 행동:** 15원칙 × 2~3 SBI 스토리(재사용 OK) + **모든 스토리 정량화** + **영어 리허설** + 60초 자기소개.
**🟢 이력서/iOS 방어:** 모든 프로젝트 방어 + Swift Concurrency/SwiftUI 복습(게이트라서가 아니라 이력서 신뢰성).
**⚙️ 로지스틱스:** 리크루터에게 **루프 언어/라운드 수/경력직 CT 여부** 확인. Zoom/Teams + HackerRank/CoderPad 사전 테스트.
