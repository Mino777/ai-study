### 시나리오 2: "코드를 짜보세요"

**질문**: "사용자 프로필 정보를 API에서 fetch해서 UI에 표시하되, 타임아웃 15초, 재시도 3회 (exponential backoff)"

**면접 전략**:
```
1️⃣ 요구사항 재확인 (10초)
"타임아웃 15초, 최대 3회 재시도가 맞나요?"

2️⃣ 타입부터 정의 (30초)
- User struct (Codable)
- APIError enum

3️⃣ 함수 뼈대 (1분)
```swift
func fetchUserProfile() async throws -> User {
    var lastError: Error?
    
    for attempt in 1...3 {
        do {
            var request = URLRequest(url: userURL)
            request.timeoutInterval = 15
            
            let (data, response) = try await URLSession.shared.data(for: request)
            
            guard let httpResponse = response as? HTTPURLResponse,
                  (200...299).contains(httpResponse.statusCode) else {
                throw APIError.invalidResponse
            }
            
            return try JSONDecoder().decode(User.self, from: data)
        } catch {
            lastError = error
            
            if attempt < 3 {
                let delay = pow(2.0, Double(attempt)) // exponential backoff: 2s, 4s
                try await Task.sleep(nanoseconds: UInt64(delay * 1_000_000_000))
            }
        }
    }
    
    throw lastError ?? APIError.unknown
}
```

4️⃣ 질문 예상 & 답변 준비
Q. "왜 JSONDecoder을 매번 생성하나?"
A. "성능 최적화를 위해 static 변수로 만들 수 있습니다."

Q. "Task.sleep은 어떤 영향이 있나?"
A. "백그라운드 스레드에서 실행하므로 UI는 차단되지 않습니다."
```
