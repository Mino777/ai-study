### /message poll Interface
```bash
/message poll <task-id> --timeout 60
# → "completed" + 결과 반환 (idempotent: 재호출 시 동일 결과)
# → "pending" (아직 실행 중) → --timeout 120으로 재시도
# → "failed" + 에러 로그 → /message retry <task-id>로 수동 재시도
# → "not_found" (task-id 없음) → /message list로 확인
```

## Implementation Phases

### Phase 1: VPS + Hermes 설치 (Day 1)
1. VPS 프로바이더 선택 (Hetzner CX22 ~$4/mo 또는 Hostinger 등)
2. VPS 초기 세팅: Node 24+, ufw 방화벽, SSH key only
3. Pre-flight 체크:
   - `node --version` (24+ 확인)
   - 디스크 여유 >5GB
   - Fallback: 스크립트 실패 시 `git clone + npm install` 수동 설치
4. 설치 스크립트 확인 후 실행 (Outside Voice 반영):
   ```bash
   curl -fsSL https://raw.githubusercontent.com/NousResearch/hermes-agent/main/scripts/install.sh -o install.sh
   less install.sh  # 내용 확인
   bash install.sh  # 확인 후 실행
   ```
   - 공식 repo: github.com/NousResearch/hermes-agent (64K+ stars, Feb 2026~)
   - 필수: 64K+ context 모델 지원
5. `hermes model` — Claude Sonnet (기본, 빠른 위임), Opus (복잡한 분석)
   - 모델 라우팅: Hermes가 작업 복잡도에 따라 자동 선택 (hermes model 설정)
6. `hermes doctor` — MUST PASS before Phase 2
7. `hermes tools` — 도구 구성

### Phase 1.5: VPS 보안 하드닝 (Day 1)
1. API 키: `.env` + `chmod 600` (git 추적 금지)
2. TLS: Let's Encrypt (Hermes 웹 대시보드용)
3. systemd 서비스 등록 (자동 재시작 + 로그 rotation)
4. 월간 API 비용 한도: $30 (초과 시 Telegram 알림)
5. Health check 크론 (6시간마다):
   ```bash
   hermes status --json | jq '.'
   # Alert if: uptime <6h, last_task >6h ago, api_error_rate >5%, disk <1GB
   # Action: systemd restart + Telegram 알림
   ```

   - /promote-solution이 Compiled Truth 섹션을 자동 prepend하도록 수정
2. /message poll 기능 구현 (messages/ 디렉토리 watch + timeout)
3. Hermes 학습 결과 → 위키 보강 파이프라인 MVP:
   - MVP: Hermes output → messages/hermes-{id}.json → /message poll → /validate-ai-output → 수동 merge
   - Hermes 자동 생성 스킬은 /validate-ai-output 거친 후에만 위키 반영
   - Future: 시맨틱 중복 감지 + 자동 병합 (Non-Scope)

### Phase 6: 2주 운영 + 데이터 수집 (Week 1-2)
1. Telegram으로 일 1+ 리서치 작업 위임 (수동 트리거)
2. 스킬 자동 생성 패턴 관찰
3. **Sampling 요건**:
   - 최소 10개 작업 완료 (단순 "14일 × 1"이 아님)
   - 다양한 유형: research + markdown 생성 + backlink 탐색
   - QA: Hermes 결과 3개 이상 /compound 결과와 직접 비교
4. **Quality Signal**:
   - 위키 스키마 준수율 >95% → Keep
   - jominho 수정 횟수 평균 <2회/결과 → Keep
5. Go/No-Go는 상단 Decision Tree 참조
6. confidence:2→3 승격 기준: 직접 설치 + 2주 운영 + 비교 데이터 확보
