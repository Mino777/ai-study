## 잘된 것

1. **딥리서치 → 즉시 구현 루프**: 한국 빅테크 면접 트렌드, 합격 후기, 학습 과학 리서치를 바로 코드로 반영. 리서치와 구현 사이 지연 0.
2. **iOS/FDE 듀얼트랙 완전 분리**: 각 탭에서 트랙 전환 시 다른 콘텐츠 표시. 데일리 팁, CS 토픽, 시스템 디자인 케이스까지 독립.
3. **Progressive Disclosure 적용**: Overview 10개 섹션 → Hero Card + Compact Stats + 아코디언. 인지 부하 대폭 감소.

## 아쉬운 것

1. **범위 추정 실패**: "면접 페이지 하나 만들자"에서 시작했지만 4,886줄까지 확장. 끊을 타이밍을 놓쳐 세션이 매우 길어짐. 2-3 Journal 단위로 끊었어야 했다.
2. **Turbopack 캐시 이슈 디버깅에 시간 소비**: Day 그리드 flex-wrap 수정이 반영 안 되는 문제에 ~30분 소비. 브라우저 캐시가 원인이었는데 서버 캐시만 의심함.
3. **테스트 부재**: 4,886줄의 새 코드에 테스트 0개. 퀴즈 셔플 로직, 플래시카드 필터링, localStorage 영속성 등 핵심 로직에 최소한의 테스트가 필요했음.
