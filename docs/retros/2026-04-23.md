## 아쉬운 것 / 판단 실수 (Anti-Rationalization 적용)

바이어스 점검: 아래는 **내 판단 실수**로 리프레이밍한 것.

1. **aidy-architect 29a7669을 스프린트 시작 시점에 감지 못 했음** — `projects-sync` 류 사전 점검을 실행 안 해서 로컬 divergence를 skillify 스프린트 한복판에서야 발견. 메모리 `feedback_projects_sync_includes_aidy.md`가 존재하는데 반사적으로 트리거 안 함. **리프레이밍**: "다른 세션 작업 때문에 divergence" 아니고 "사전 점검을 의식적으로 스킵".
2. **check-skills-reachable 첫 정규식 버그 (`## Skill routing` 인라인 매치)** — CLAUDE.md에 방금 추가한 본문에 "`## Skill routing`" 문자열 인라인 코드를 넣고 바로 스크립트 돌렸더니 거기 매치. 줄 단위 파싱으로 5분 만에 수정했지만, **자기 프로젝트 dogfood 한 번만 돌렸어도 작성 중에 잡았을 것**. "unit test만 통과하면 된다"고 판단한 게 오류 — 실데이터 테스트 먼저.
3. **KPI "moneyflow dispatch 토큰 절감" 미측정 재방치** — 지난 회고(2026-04-22)에서도 "적용됨 (미측정)"으로 썼던 걸 이번에도 똑같이 답습. ccusage 래퍼나 비교 스크립트를 이번에 만들 기회였는데 스코프 추가 판단 실패. **리프레이밍**: "시간 부족" 아니고 "측정 인프라 우선순위 판단 실패".
4. **vitest 첫 실행에서 전원 실패 원인(=`/dev/null` 경로 처리)을 한 번에 못 잡음** — error message 안 보고 재시도 → 시간 낭비 1분. Vitest 실패 시 `beforeAll` 단계부터 체크하는 반사행동 필요.
