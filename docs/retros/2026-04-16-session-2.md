## 잘된 것

- **실측 기반 스케일링 진단**: "100 엔트리면 큰일 나나?" 같은 추정 질문에 `.next/server/app/*.rsc` 를 gzip 해서 페이지별 페이로드를 직접 측정. 결과: 엔트리 수 자체보단 **client에 새는 layout props** 가 진짜 병목이라는 걸 숫자로 증명
- **fix 되돌아보기 습관**: android 수정 후 "같은 패턴이 iOS/backend 에도 있을 것"을 예측하고 grep으로 확인 → 추가 19곳 발견. 이전 commit `ed34d26` 이 "subgraph만 수정"으로 멈췄던 것이 교훈

- **RSC payload 대시보드 -70% 감축**: 대시보드는 자체 `<SearchDialog>` + `<GraphSearchProvider>` mount 가 있어서 중복 직렬화로 ~50KB → 15KB. 예상보다 큰 효과
