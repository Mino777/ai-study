# /validate-mdx — MDX/Mermaid 문법 함정 사전 검증

MDX 콘텐츠 작성 또는 AI 생성 후, 커밋 전에 실행.
5가지 반복 함정 패턴을 검증하여 빌드/런타임 에러를 방지한다.

> 출처: docs/solutions/mdx/ 5건 (2026-04-09 ~ 04-16) 누적 패턴
> dry-run 검증: 2026-04-17 세션 10에서 127 MDX 전수 스캔. 초판 grep false positive 100% → 코드블록 제외 로직 추가

---

## 입력

대상 파일 경로. 미지정 시 `git diff --name-only --cached` 또는 최근 변경된 `.mdx` 파일.

---

## 핵심 원칙: 코드블록 제외

단순 grep은 코드블록(``` ... ```) 안의 패턴을 전부 잡아서 false positive 100%.
**반드시 코드블록 밖의 텍스트만 검사해야 한다.**

검증 방법: 대상 파일을 Read로 읽고, 코드블록(``` 또는 ~~~) 사이의 줄은 건너뛰면서 아래 5개 패턴을 확인한다.
grep으로는 코드블록 내/외 구분이 불가능하므로, **파일을 읽고 수동 확인**하는 방식을 사용한다.

---

## 5 함정 검증 (순서대로)

### 함정 1: `<br>` self-closing 누락

MDX는 JSX 파서 → `<br>` (HTML void) 컴파일 에러.

**탐지**: 코드블록 밖에서 `<br>` 또는 `<br >` (self-closing `/` 없음)
**제외**: 코드블록 안, 백틱 안, 텍스트에서 패턴을 *설명*하는 문장
**수정**: `<br>` → `<br />`

### 함정 2: Mermaid subgraph 이름 공백

`subgraph Standard RAG` → `Standard`만 인식, `RAG`는 garbage.

**탐지**: mermaid 코드블록 안에서 `subgraph <단어> <단어>` (id 없이 공백 포함 이름)
**수정**: `subgraph id ["Label with spaces"]`

### 함정 3: JSX 파싱 함정

코드블록 **밖**의 본문 텍스트에서만 검사:
- `{...}` — JSX expression으로 해석됨 → 백틱 래핑
- `<숫자` (예: `<3`) — JSX 태그 시작으로 해석 → 자연어 대체
- HTML void element (`<col>`, `<hr>` 등) — self-closing 필요

**제외 (false positive 주범)**:
- 코드블록 안의 모든 `{}`
- 인라인 코드(백틱) 안의 `{}`
- JSX/TSX 컴포넌트 코드 (의도적 사용)

### 함정 4: Mermaid 노드 라벨 특수문자 따옴표 누락

빌드는 통과하지만 **런타임 Mermaid 렌더 실패** (가장 위험).

**탐지**: mermaid 코드블록 안에서 `[라벨]` 또는 `{라벨}` 형태의 노드 중, 라벨에 `<br/>`, `→`, `<`, `>` 포함인데 따옴표로 안 감싼 것
**수정**: `Node["label<br/>text"]` — 따옴표로 감싸기

### 함정 5: Mermaid cylinder `[("...")]` 중첩 괄호

**탐지**: mermaid 코드블록 안에서 `[("...")]` 구문 중 라벨에 괄호 포함
**수정**: cylinder `[("...")]` 대신 일반 노드 `["..."]` 사용

## 실행 흐름

1. 대상 파일 목록 확정
2. 각 파일을 Read로 읽기
3. 코드블록(``` 경계)을 파악하고, 블록 내/외를 구분
4. 함정 1, 3: 코드블록 **밖** 텍스트만 검사
5. 함정 2, 4, 5: mermaid 코드블록 **안**만 검사
6. 발견 시 파일명:줄번호:패턴 출력 + 자동 수정 제안
7. `npm run build` 로 최종 확인

## false positive 가이드 (dry-run 학습)

| 패턴 | 흔한 false positive | 판별법 |
|---|---|---|
| `<br>` | 텍스트에서 패턴을 *설명*하는 문장 | 주변에 "태그", "에러" 등 메타 표현 있으면 무시 |
| `{...}` | 코드블록 안 JS/TS 코드 | ``` 경계 안이면 무시 |
| `<숫자` | Mermaid 라벨 안의 비교 (`<1m`) | 이미 따옴표로 감싸져 있으면 무시 |
| `→` in `[]` | 마크다운 테이블 셀 | mermaid 블록 밖이면 무시 |
