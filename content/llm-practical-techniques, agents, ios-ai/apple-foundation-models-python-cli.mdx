> 이 엔트리는 Blake Crosley의 [Foundation Models from Python: the fm CLI](https://blakecrosley.com/blog/foundation-models-python-fm-cli)을 정독하고 핵심을 추출한 것이다.

Apple의 온디바이스 파운데이션 모델이 Swift의 벽을 허물고 커맨드 라인(`fm` CLI)과 Python SDK를 통해 개방되었습니다. 이는 별도의 앱 빌드나 API 키 없이, macOS에 내장된 LLM을 스크립팅, 자동화, 평가 파이프라인에 직접 통합할 수 있는 강력한 패러다임 전환입니다. 이제 로컬 환경에서 무료로, 네트워크 지연 없이 LLM을 활용한 자동화가 가능해졌습니다.

이 변화의 핵심은 Apple이 WWDC26 세션 334에서 발표한 두 가지 새로운 인터페이스입니다.
1.  **`fm` CLI**: macOS 27에 사전 설치된 커맨드 라인 도구
2.  **Foundation Models SDK for Python**: `pip`으로 설치 가능한 Python 패키지

이제 개발자와 ML 엔지니어는 Xcode 프로젝트 없이도 셸 스크립트나 Jupyter 노트북에서 Apple의 온디바이스 모델과 더 강력한 Private Cloud Compute 모델을 손쉽게 호출할 수 있습니다.

### 왜 중요한가: 로컬 LLM의 스크립트화

가장 큰 변화는 **접근성**입니다. 이전까지 Apple의 온디바이스 모델은 Xcode에서 Swift로 컴파일된 앱 내부에서만 `LanguageModelSession`을 통해 접근 가능했습니다. 이로 인해 간단한 프롬프트 테스트나 자동화 스크립트에 활용하기 어려웠습니다.

`fm` CLI와 Python SDK는 이 장벽을 제거하여 다음과 같은 새로운 가능성을 엽니다.
- **비용 없는 자동화**: API 비용이나 사용량 제한 없이 로컬 파일 정리, 텍스트 분류 등의 작업을 자동화하는 셸 스크립트를 작성할 수 있습니다.
- **신속한 프로토타이핑**: 아이디어를 검증하기 위해 전체 앱을 빌드할 필요 없이, 터미널에서 `fm respond` 명령으로 즉시 모델의 반응을 확인할 수 있습니다.
- **체계적인 평가**: Python의 풍부한 데이터 과학 생태계(Pandas, Matplotlib 등)를 활용하여 여러 프롬프트 변형에 대한 모델의 성능을 정량적으로 평가하고 비교하는 파이프라인을 구축할 수 있습니다.

이 모든 것이 Apple Silicon Mac의 로컬 환경에서 실행되므로, 보안과 속도 면에서 큰 이점을 가집니다.

### 핵심 패턴 1: `fm` CLI를 활용한 구조화된 출력 자동화

가장 실용적인 패턴은 `fm respond` 명령어와 `fm schema`를 결합하여 **예측 가능한 JSON 출력**을 얻고 이를 셸 스크립트에서 활용하는 것입니다. LLM의 자유로운 텍스트 응답을 파싱하는 것은 불안정하며 스크립트를 깨지기 쉽게 만듭니다. 구조화된 출력은 이 문제를 해결합니다.

WWDC 세션에서 시연된 "파일 분류 스크립트"가 이 패턴의 대표적인 예시입니다.
1.  **목표**: 폴더 내 파일들을 이름만 보고 '최종본'과 '초안'으로 분류하여, 최종본은 백업하고 초안은 아카이브로 옮긴다.
2.  **Schema 정의**: `fm schema object`를 사용해 `finals`와 `drafts`라는 두 개의 문자열 배열을 갖는 JSON 구조를 정의합니다.
3.  **모델 호출**: `fm respond`에 파일 목록을 프롬프트로 전달하고, `--schema` 옵션으로 앞서 정의한 스키마를 지정합니다.
4.  **결과 처리**: 모델은 정의된 스키마에 맞춰 JSON을 반환합니다. 스크립트는 `jq` 같은 도구로 이 JSON을 파싱하여 파일 이동/복사 작업을 수행합니다.

이 워크플로는 Swift의 'guided generation' 기능을 커맨드 라인으로 가져온 것으로, "모델의 판단에 따라 결정론적인 작업을 수행하는" 모든 자동화에 적용할 수 있습니다.

### 핵심 패턴 2: Python SDK를 이용한 모델 평가 및 연동

Python SDK는 머신러닝 엔지니어와 데이터 과학자를 겨냥합니다. Swift 프레임워크와 거의 동일한 API(`LanguageModelSession`, `respond` 등)를 제공하여 Python의 강력한 생태계와 온디바이스 모델을 연결합니다.

설치를 위해서는 Python 3.10+, Xcode, 그리고 Apple Silicon Mac이 필요하며, 이는 SDK가 외부 API를 호출하는 것이 아니라 macOS에 내장된 네이티브 모델의 바인딩임을 시사합니다.

```python
// 예시: Python SDK를 사용한 프롬프트 응답 (원문 기반 재구성)
import foundationmodels as fm

// 1. 세션 생성 (Swift의 LanguageModelSession과 유사)
// Private Cloud Compute 모델을 사용하려면 model 인자 추가
session = fm.LanguageModelSession(instructions="너는 유용한 AI 어시스턴트야.")

// 2. 프롬프트에 응답 요청
prompt = "Apple의 새로운 'fm' CLI의 가장 큰 장점은 무엇인가요?"
result = session.respond(prompt)

// 3. 결과 출력
print(result.output)
```

이 SDK의 진정한 가치는 평가 파이프라인 구축에 있습니다. 예를 들어, 수십 개의 프롬프트 변형을 데이터셋에 대해 실행하고, 그 결과를 Pandas DataFrame으로 정리한 뒤, 정답률이나 특정 기준에 따른 성능을 Matplotlib으로 시각화하여 최적의 프롬프트를 찾아낼 수 있습니다.

### 실전 적용: `ai-study` 위키 콘텐츠 생성 도우미 개발

`ai-study` 프로젝트에서 LLM을 활용해 기술 블로그나 논문을 요약하고 핵심 패턴을 추출하는 기능을 개발할 때, Apple의 새로운 도구들을 다음과 같이 활용할 수 있습니다.

1.  **초기 프롬프트 탐색 (`fm` CLI)**:
    - `ai-study`의 새로운 위키 엔트리를 작성하기 전에, 원문 아티클의 URL이나 텍스트를 `fm respond`에 파이프하여 다양한 요약 프롬프트를 빠르게 테스트합니다.
    - 예: `cat article.txt | fm respond "이 글의 핵심 주장 세 가지를 불렛 포인트로 요약해줘."`
    - 이를 통해 Xcode 프로젝트나 Python 스크립트 없이도 최적의 프롬프트 구조를 신속하게 탐색할 수 있습니다.

2.  **구조화된 정보 추출 (`fm schema`)**:
    - `ai-study` 엔트리에 필요한 '핵심 개념', '주요 인물', '참고 자료' 등의 필드를 `fm schema object`로 정의합니다.
    - `fm respond --schema`를 사용하여 원문에서 이 정보들을 JSON 형태로 안정적으로 추출하는지 검증합니다. 이 JSON은 위키 엔트리의 초안을 자동으로 생성하는 데 사용될 수 있습니다.

3.  **품질 평가 파이프라인 (Python SDK)**:
    - 가장 효과적인 프롬프트 후보군이 정해지면, Python 스크립트를 작성하여 100개의 `ai-study` 소스 아티클에 대해 일괄적으로 실행합니다.
    - Python SDK를 사용하여 각 아티클로부터 구조화된 데이터를 추출하고, `pandas`를 이용해 결과를 저장합니다.
    - "핵심 개념 필드가 비어있는 비율", "참고 자료 URL의 유효성" 등 자체적인 품질 지표를 설정하고, 어떤 프롬프트가 가장 높은 점수를 받는지 정량적으로 분석하여 최종 프롬프트를 확정합니다. 이 과정은 모두 로컬 Mac에서 API 비용 없이 수행됩니다.
