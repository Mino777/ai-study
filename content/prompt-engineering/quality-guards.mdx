## 핵심 개념

LLM은 본질적으로 **낙관 편향**(긍정적 답변 선호)과 **환각**(사실이 아닌 정보 생성) 경향이 있다. 품질 가드는 시스템 프롬프트에 주입하는 규칙으로, 이런 경향을 억제한다.

## 실전 품질 가드 패턴

mino-moneyflow 프로젝트에서 검증된 6가지 가드:

### 1. 베이스레이트 주입 (Base Rate Injection)

모델에게 현실의 기저 확률을 알려주는 것. "대부분의 주식이 시장을 이긴다"는 환상을 깬다.

```
"주식의 약 20%만 시장 수익률을 초과합니다.
'강력 매수' 신호의 정확도는 약 55%입니다.
이 사실을 판단의 기준으로 삼으세요."
```

### 2. 반낙관 편향 가드 (Counter-Positivity Bias)

LLM의 문서화된 낙관 편향에 적극적으로 저항하는 지시.

```
"당신은 긍정적 답변을 선호하는 경향이 있습니다.
의식적으로 부정적 시나리오에 동등한 비중을 두세요.
'기회'를 언급할 때마다 동등한 '위험'도 제시하세요."
```

### 3. 정량적 앵커링 (Quantitative Anchoring)

모든 주장에 구체적 숫자를 요구하여 모호한 긍정을 방지.

```
"모든 주장에는 최소 1개의 구체적 숫자가 포함되어야 합니다.
'성장 가능성이 높다' → '연간 매출 성장률 23% (YoY)'"
```

### 4. 인용 태깅 (Citation Tagging)

출처를 명시하여 환각 여부를 추적 가능하게 만듦.

```
"모든 문장에 출처 태그를 붙이세요:
[DATA] — 제공된 실제 데이터에서 도출
[CALC] — 계산으로 도출
[EST]  — 추정 또는 의견"
```

### 5. 프리모템 분석 (Pre-Mortem Analysis)

예측이 실패했다고 가정하고 이유를 먼저 분석하게 하는 기법. 과신을 크게 줄인다.

```
"3개월 후 당신의 예측이 틀렸다고 가정하세요.
가장 가능성 높은 실패 원인 3가지를 먼저 작성하세요."
```

### 6. 강철 논증 의무 (Steelman Obligation)

상대방의 가장 강한 논거를 먼저 제시한 후에야 반박할 수 있는 규칙.

```
"반대 의견을 반박하기 전에,
먼저 반대 측의 가장 강력한 논거를 정확하게 기술하세요."
```

## 가드 주입 패턴

시스템 프롬프트에 자동으로 가드를 주입하는 아키텍처:

```typescript
// 베이스 가드 + 에이전트별 커스텀 프롬프트 조합
const systemMessage = options?.agentRole
  ? `${COMMON_QUALITY_GUARDS}\n\n${options.systemMessage}`
  : options?.systemMessage;
```

모든 에이전트가 공통 가드를 공유하고, 각자의 역할별 프롬프트가 추가됨.

## 효과

- 프리모템: 과신 편향 20-30% 감소 (학술 연구 기반)
- 인용 태깅: 환각 추적 가능, 사후 검증 용이
- 강철 논증: 허수아비 논증 방지, 논리적 깊이 향상
- 베이스레이트: "모든 게 좋아 보이는" 출력 빈도 감소

## 주의점

- 가드가 너무 많으면 모델의 유용한 출력까지 억제할 수 있음
- 도메인에 맞는 베이스레이트를 정확하게 제공해야 효과적
- 가드 효과는 모델 크기에 비례 (소형 모델에서는 지시를 무시할 수 있음)

## AI Agent Directive

**Trigger**: AI 에이전트가 프로덕션용 시스템 프롬프트를 작성하거나, LLM 출력의 환각·편향을 줄여야 할 때
**Prerequisites**: [Chain-of-Thought](/wiki/prompt-engineering/chain-of-thought), [AI 출력 Zod 검증 패턴](/wiki/harness-engineering/ai-output-zod-validation-pattern)

### Actionable Steps
1. 현재 시스템 프롬프트에 품질 가드가 몇 개 있는지 확인한다
2. 프리모템 가드("이 분석이 틀렸다면 어떤 이유일까?") 추가를 검토한다
3. 인용 태깅 가드(주장마다 출처 표시 강제)로 환각 추적성을 확보한다
4. 도메인 베이스레이트를 프롬프트에 명시한다 (예: "주식 추천의 성공률은 통상 50~55%")
5. 가드 개수는 3~5개로 제한하고, 모델 출력 품질이 떨어지면 축소한다
6. Multi-Agent 파이프라인에서는 공통 가드를 분리하여 모든 에이전트에 주입한다

### Anti-patterns
- 가드를 10개 이상 쌓아서 모델의 유용한 출력까지 억제하는 것
- 도메인 베이스레이트 없이 "정확하게 답해줘"만 쓰는 것
- 소형 모델에서 복잡한 가드를 기대하는 것 (지시 무시 확률 높음)
