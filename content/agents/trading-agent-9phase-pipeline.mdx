## 갭 / 맥락

단일 LLM에게 "이 주식을 분석해줘"라고 하면 자신감 있는 답을 내놓지만, 그 답이 얼마나 신뢰할 수 있는지 알 방법이 없다. LLM은 확증 편향에 취약하고, 자기 답변의 약점을 스스로 인식하지 못한다.

moneyflow의 해법: **단일 에이전트가 결론을 내리는 게 아니라, 13개 에이전트가 적대적으로 토론하는 구조 자체가 환각을 줄인다.**

---

## 핵심 패턴 — 9-Phase Pipeline

```
Phase 1: DATA COLLECTION (4 병렬 에이전트)
├── Market Analyst      ← 기술 지표 (RSI, MACD, 볼린저)
├── Fundamentals Analyst ← 재무제표 (PER, PBR, ROE)
├── News Analyst        ← 뉴스 감성 분석
└── Macro Analyst       ← 매크로 경제 지표

Phase 2: ADVERSARIAL DEBATE (2 병렬 에이전트)
├── Bull Advocate       ← 매수 논거 (Steelman 규칙)
└── Bear Advocate       ← 매도 논거 (Steelman 규칙)

Phase 3: JUDGMENT (1 에이전트)
└── Investment Judge    ← 토론 중재, conviction 점수

Phase 4: RISK ASSESSMENT (2 병렬 에이전트)
├── Risk Analyst        ← 정량적 리스크 평가
└── Devil's Advocate    ← 반대 시나리오 강제 제시

Phase 5: RISK CALIBRATION (1 에이전트)
└── Risk Calibrator     ← 리스크 통합 + confidence 보정

Phase 6-9: EXECUTION (후처리)
├── Portfolio Manager   ← 포지션 사이징 + 실행 계획
├── CIO Review         ← 최종 검토
├── Memory Integration ← BM25로 과거 유사 사례 주입
└── Quality Judge      ← LLM-as-Judge 4차원 채점
```

### Phase 2 상세: Steelman 규칙

```
[Bull Advocate 프롬프트]
규칙: 반박하기 전에 반드시 Bear 측의 가장 강한 논거를
직접 인용하고 그 강도를 인정한 후에만 반박을 시작하라.

예시:
"Bear 측이 지적한 부채비율 200% 초과는 실제로 동종 업계 평균(150%)을
크게 넘어서는 수치이며 이 리스크는 유효합니다.
그러나 이를 상쇄하는 요인으로..."
```

### Tier별 에이전트 구성

| Tier | 에이전트 수 | 토론 라운드 | 모델 |
|------|-----------|-----------|------|
| Free | 7 (경량) | 1라운드 | Gemini Flash |
| Pro | 13 (전체) | 3라운드 | Gemini Pro + Claude Sonnet |
| Premium | 13 + Deep Think | 3라운드 | Claude Sonnet (전체) |

### 각 에이전트의 출력 표준

```typescript
interface AgentReport {
  signal: "STRONG_BUY" | "BUY" | "NEUTRAL" | "SELL" | "STRONG_SELL";
  confidence: number;  // 0-100
  summary: string;
  key_points: string[];
}
```

모든 에이전트가 동일한 인터페이스를 반환 → 파이프라인 어디서든 조합/비교 가능.

## 운영 데이터

- **일일 비용 상한**: $5/day (13 에이전트 × 평균 $0.04/회 × ~30회/day)

## AI Agent Directive

### Trigger
- 복잡한 판단이 필요한 AI 파이프라인에서 단일 에이전트의 편향을 줄이고 싶을 때
- 멀티에이전트 토론 구조를 설계할 때

### Prerequisites
- [Multi-Agent Pipeline](/wiki/agents/multi-agent-pipeline)
- [Multi-Provider Circuit Breaker](/wiki/agents/multi-provider-circuit-breaker)

### Actionable Steps
1. **역할별 에이전트 분리** — 데이터 수집/분석/판단/리스크를 각각 독립 에이전트로
2. **적대적 토론 구조** — Bull/Bear + Steelman 규칙으로 허수아비 논증 차단
3. **Devil's Advocate** — 최종 판단 직전에 반대 시나리오 강제
4. **표준 출력 인터페이스** — signal + confidence + summary + key_points
5. **Memory Loop** — 과거 사례 주입 + 앵커링 방지 프롬프트
6. **Confidence Calibration** — 3 버킷 단조성 검사로 자신감 vs 실제 적중률 비교

### Anti-patterns
- **토론 없는 직렬 파이프라인** — 확증 편향 누적. 앞 에이전트의 결론을 뒤 에이전트가 무비판적으로 수용
- **Steelman 없는 토론** — LLM이 상대의 약한 주장만 공격하는 허수아비 논증
- **무제한 토론 라운드** — 비용 폭주. 3라운드가 비용/품질 균형점 (moneyflow 경험칙)
