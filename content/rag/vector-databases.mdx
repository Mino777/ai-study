## 핵심 개념

Vector Database는 고차원 벡터(임베딩)를 저장하고 **유사도 기반 검색**을 수행하는 전문 데이터베이스다. RAG 파이프라인에서 검색 단계의 핵심 인프라.

## 왜 일반 DB가 아닌 Vector DB인가

일반 DB는 exact match에 최적화되어 있다. "이 벡터와 가장 비슷한 벡터 100개를 찾아라"는 질의에는 전혀 다른 인덱싱 전략이 필요하다. ANN(Approximate Nearest Neighbor) 알고리즘이 핵심.

## 주요 ANN 알고리즘

- **HNSW** (Hierarchical Navigable Small World): 가장 널리 쓰임. 높은 recall + 빠른 검색.
- **IVF** (Inverted File Index): 클러스터 기반. 대규모 데이터셋에 적합.
- **PQ** (Product Quantization): 메모리 효율적. 정확도 트레이드오프.

## 주요 제품 비교

| 제품 | 타입 | 특징 |
|------|------|------|
| Pinecone | Managed | 가장 쉬운 시작, 서버리스 옵션 |
| Weaviate | Self-hosted/Cloud | 하이브리드 검색 (벡터+키워드) |
| Chroma | Embedded | 로컬 개발에 최적, Python 네이티브 |
| Qdrant | Self-hosted/Cloud | Rust 기반, 높은 성능 |
| pgvector | Extension | PostgreSQL 확장, 기존 스택 활용 |

## 선택 기준

1. **규모**: 백만 벡터 이하면 pgvector로 충분
2. **운영 부담**: 관리형 원하면 Pinecone, 직접 운영이면 Qdrant
3. **하이브리드 검색**: 벡터+키워드 필요하면 Weaviate
4. **프로토타입**: Chroma로 시작, 필요 시 마이그레이션
