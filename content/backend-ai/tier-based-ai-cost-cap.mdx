## 요약

AI API를 서비스에 통합할 때 "이번 달 비용이 얼마나 나왔는지" 실시간으로 모르면 과금 폭탄을 맞는다. 이 패턴은 **티어별 월간 예산 상한**을 설정하고, **인메모리 누적 트래커**로 실시간 추적하며, **80% 경고 + 100% 차단** 2단계 게이트로 방어한다.

## 아키텍처

```
API 요청 → checkBudgetBeforeAnalysis() → [ok] → 에이전트 실행
                                        → [warning] → 에이전트 실행 + 경고 메시지
                                        → [blocked] → 429 응답 (즉시 반환)
                                              ↓
                                        에이전트 실행 완료
                                              ↓
                                        accumulateCost() → MonthlyAccumulator 업데이트
```

## 1. 티어별 예산 정의

```typescript
const TIER_BUDGET_USD: Record<Tier, number> = {
  free:    0.50,   // $0.50/월 — 체험용
  pro:     15.00,  // $15/월 — 일반 사용
  premium: 50.00,  // $50/월 — 헤비 사용
};

// 분석 1회당 예상 비용 (사전 체크용)
const ESTIMATED_COST_PER_ANALYSIS: Record<Tier, number> = {
  free:    0.08,   // Gemini only
  pro:     0.35,   // 혼합 모델
  premium: 0.60,   // Claude 포함
};
```

## 2. 인메모리 누적 트래커

```typescript
class MonthlyAccumulator {
  private month: string;           // "2026-04"
  private byUser: Map<string, number>;  // userId → 누적 USD

  accumulateCost(userId: string, costUsd: number): void {
    this.autoResetIfNewMonth();
    const current = this.byUser.get(userId) ?? 0;
    this.byUser.set(userId, current + costUsd);
  }

  getSpent(userId: string): number {
    this.autoResetIfNewMonth();
    return this.byUser.get(userId) ?? 0;
  }

  private autoResetIfNewMonth(): void {
    const now = new Date().toISOString().slice(0, 7); // "2026-04"
    if (now !== this.month) {
      this.month = now;
      this.byUser.clear();  // 월 전환 시 자동 리셋
    }
  }
}

// 싱글톤 — 서버리스 인스턴스당 1개
const accumulator = new MonthlyAccumulator();
```

**왜 인메모리인가?** 이 트래커는 **2차 보조 방어선**이다. 1차 방어는 티어별 요청 횟수 제한(rate limiting). Vercel warm restart 시 리셋되지만, 1차 방어가 있으므로 비용 폭주는 이미 막힌 상태. 정밀도보다 속도(non-blocking fire-and-forget)를 우선한다.

## 3. 예산 게이트: 2단계 체크

```typescript
interface BudgetCheckResult {
  status: "ok" | "warning" | "blocked";
  spentUsd: number;
  budgetUsd: number;
  usagePercent: number;
  message?: string;
}

function checkBudget(tier: Tier, spentUsd: number): BudgetCheckResult {
  const budget = TIER_BUDGET_USD[tier];
  const percent = (spentUsd / budget) * 100;

  if (percent >= 100) {
    return {
      status: "blocked",
      spentUsd, budgetUsd: budget, usagePercent: percent,
      message: `월간 AI 예산 $${budget} 초과. 다음 달에 리셋됩니다.`,
    };
  }

  if (percent >= 80) {
    return {
      status: "warning",
      spentUsd, budgetUsd: budget, usagePercent: percent,
      message: `월간 AI 예산 ${percent.toFixed(0)}% 사용 중 ($${spentUsd.toFixed(2)}/$${budget}).`,
    };
  }

  return { status: "ok", spentUsd, budgetUsd: budget, usagePercent: percent };
}
```

## 4. 사전 체크 (Pre-flight)

```typescript
function checkBudgetBeforeAnalysis(userId: string, tier: Tier): BudgetCheckResult {
  const spent = accumulator.getSpent(userId);
  const estimated = ESTIMATED_COST_PER_ANALYSIS[tier];
  // 현재 누적 + 예상 비용으로 사전 판단
  return checkBudget(tier, spent + estimated);
}
```

실행 **전에** 예상 비용을 더해 체크. 실행 후 실제 비용이 다르더라도 큰 오차는 아니다.

## 5. 비용 계산: 토큰 → USD

```typescript
interface TokenUsage {
  inputTokens: number;
  cachedTokens: number;   // 90% 할인 적용
  outputTokens: number;
}

function calculateCost(usage: TokenUsage, provider: Provider): number {
  const pricing = PROVIDER_PRICING[provider];
  const inputCost = (usage.inputTokens - usage.cachedTokens) * pricing.input;
  const cacheCost = usage.cachedTokens * pricing.input * 0.1;  // 90% 할인
  const outputCost = usage.outputTokens * pricing.output;
  return inputCost + cacheCost + outputCost;
}
```

## 6. 라우트 핸들러 통합

```typescript
// POST /api/analyze
export async function POST(req: Request) {
  const { userId, tier } = await authenticate(req);
  
  // 1. 사전 예산 체크
  const budget = checkBudgetBeforeAnalysis(userId, tier);
  if (budget.status === "blocked") {
    return Response.json({ error: budget.message }, { status: 429 });
  }

  // 2. 에이전트 실행
  const result = await runAnalysis(userId, tier);

  // 3. 비용 누적 (fire-and-forget)
  accumulator.accumulateCost(userId, result.costUsd);

  // 4. 응답 (경고 포함)
  return Response.json({
    data: result.data,
    ...(budget.status === "warning" && { budgetWarning: budget.message }),
  });
}
```

## 설계 트레이드오프

| 결정 | 선택 | 대안 | 이유 |
|------|------|------|------|
| 저장소 | 인메모리 | DB/Redis | 2차 방어선이므로 속도 우선. restart 리셋 허용 |
| 체크 시점 | 사전(pre-flight) | 사후(post-flight) | 초과 후 차단보다 초과 전 방지가 UX 우수 |
| 임계값 | 80%/100% 2단계 | 단일 100% | 경고 단계가 사용자에게 대비 시간 제공 |
| 리셋 주기 | 월별 자동 | 수동 리셋 | `autoResetIfNewMonth()`로 운영 부담 제거 |

## 핵심 교훈

1. **방어선은 레이어로** — Rate Limiting(1차) + 비용 누적(2차) + 모델 다운그레이드(3차, AgentCompiler). 한 겹이 뚫려도 다음이 막는다
2. **인메모리로 시작** — 정밀한 DB 추적은 트래픽이 충분히 쌓인 후. 초기에는 속도와 단순함이 더 가치있다
3. **사전 체크 + 예상 비용** — 실행 후 차단은 이미 비용이 발생한 후. 사전에 예상 비용을 더해 판단
4. **프롬프트 캐싱과 시너지** — cachedTokens 90% 할인은 비용 상한 내에서 더 많은 요청을 처리 가능하게 함
