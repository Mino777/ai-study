## Session 11 — Stall Detection 프로토콜

### Worker Stall Detection 4단계

s11에서 **Enter flush 인시던트**가 다시 발생했다. architect-cli.sh가 tmux send-keys로 워커에게 명령을 보내는데, 워커가 응답하지 않는 stall 상태에 빠짐. 이를 체계화한 4단계 프로토콜:

| Stage | 이름 | 방법 | 해결 비율 |
|-------|------|------|----------|
| 1 | 조기 확인 | 실행 마커 감지 (5회 재시도) | ~90% |
| 2 | tmux 진단 | `tmux capture-pane` 로 워커 상태 확인 | ~5% |
| 3 | 원격 개입 | tmux send-keys로 직접 명령 주입 | ~4% |
| 4 | Architect 직접 수정 | 워커 세션 종료 + Architect가 직접 코드 수정 | ~1% |

**핵심 원칙**: 항상 Stage 1부터 시작. Stage 3까지 가는 건 iOS tuist test 루프처럼 워커 자체 해결 불가능한 구조적 문제일 때만.

### Enter flush 패치

architect-cli.sh에 3가지 방어 추가:

1. **5회 재시도** — send-keys 후 실행 마커가 나타날 때까지 대기
2. **실행 마커 감지** — 워커 출력에서 특정 패턴 매칭
3. **경고 출력** — 5회 초과 시 Stage 2 에스컬레이션 안내

### Feature Sprint 패턴

s11-s13에서 반복된 구조:

```
api-contract 버전업 → server 구현 + 테스트 → iOS/Android 2-way dispatch → Gate
```

이 4단계가 **세션 단위로 반복** 가능한 이유:
- api-contract가 3-client의 단일 진실 소스 ([참조](/wiki/context-engineering/api-contract-as-3-client-source-of-truth))
- server-first → client 2-way 순서가 고정 (API 변경 포함이므로)
- Gate가 세션 내 피드백 루프 완성

## Session 14 — 3-way 동시 Dispatch + 테스트 대폭 추가

### 왜 3-way가 가능했나

s14의 WO-030/031/032는 모두 **기존 기능에 테스트 추가**:
- Server: E2E 7건 (v0.3~v0.4 엔드포인트)
- iOS: Feature 테스트 95건 (editMemory/togglePin/search/delete/export/profile)
- Android: ViewModel 테스트 75건 (동일 6기능)

**API 변경 0** → 3 워커가 서로 의존 없음 → 동시 dispatch 가능.

이전까지의 dispatch 모드:

| 세션 | 모드 | 이유 |
|------|------|------|
| s3-s4 | 2-way (server+client) | API 변경 포함 |
| s8 | send-seq (직렬) | billing 차단 진단 필요 |
| s11-s13 | server-first → 2-way | API 변경 포함 |
| **s14** | **3-way 동시** | API 변경 없음, 독립 작업 |

### 테스트 +177건

| 프로젝트 | s13 종료 | s14 추가 | s14 종료 |
|---------|---------|---------|---------|
| server | 250 | +6 E2E | 256 |
| ios | 166 | +25 | 191 |
| android | 170 | +20 | 190 |
| **합계** | **586** | **+51 (unit)** | **637** |

*iOS Feature 95건 + Android ViewModel 75건은 기존 테스트 프레임워크 내 추가 (합계에 별도 반영되지 않은 테스트 포함).*

### 인시던트

1. **Android 429** — CLI backoff 자동 대응 (기존 패턴)
2. **iOS tuist test 루프** — Stage 3 개입 (반복 패턴, 근본 해결 P2)
