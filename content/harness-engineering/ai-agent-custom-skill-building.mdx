이 엔트리는 Blake Crosley의 [Claude Code Skills: Build Custom Auto-Activating Extensions](https://crosley.blake.ai/writing/claude-code-skills/)를 정독하고 핵심을 추출한 것이다.

### 왜 중요한가: 반복적인 컨텍스트 주입의 자동화

개발자는 매번 새로운 세션에서 동일한 보안 체크리스트, 팀의 코딩 스타일 가이드, API 설계 패턴 같은 '도메인 지식'을 LLM에게 반복적으로 설명한다. 이 과정은 비효율적이며, 사람이 매번 기억에 의존해야 하는 실수의 여지를 남긴다.

Claude Code의 '스킬(Skill)'은 이 문제를 해결한다. 스킬은 사용자가 명시적으로 호출하지 않아도, LLM이 대화의 맥락을 이해하여 **자동으로 발견하고 적용하는(model-invoked)** 확장 기능이다. 이는 반복적인 프롬프트 엔지니어링을 넘어, LLM에게 영구적인 도메인 전문성을 부여하는 패러다임의 전환을 의미한다. "매번 알려주는" 방식에서 "알아서 알게 하는" 방식으로의 변화다.

### 핵심 패턴

#### 1. `description` 기반의 의미론적 활성화

스킬의 핵심은 `SKILL.md` 파일의 YAML frontmatter에 있는 `description` 필드다. Claude는 단순한 키워드 매칭이 아니라, **LLM 추론**을 통해 사용자의 요청이 `description`의 의미와 일치하는지를 판단하여 스킬을 활성화한다.

-   **잘못된 예시**: `"git rebase helper"` (너무 광범위하여 `git status` 같은 관련 없는 명령어에도 활성화될 수 있음)
-   **좋은 예시**: `"Review code for security vulnerabilities, performance issues, and best practice violations. Use when examining code changes, reviewing PRs, analyzing code quality, or when asked to review, audit, or check code."` (구체적인 시나리오와 트리거 단어를 명시하여 활성화 시점을 명확히 함)

이 `description`은 스킬의 'API' 역할을 하므로, 언제, 왜 이 스킬이 필요한지를 명확하고 상세하게 기술하는 것이 가장 중요하다.

#### 2. 결정 프레임워크: Skill vs. Command vs. Subagent

모든 반복 작업을 스킬로 만들면 안 된다. 저자의 `git-rebase-helper` 스킬 실패 사례처럼, 너무 자주 활성화되는 스킬은 오히려 컨텍스트를 오염시킨다. 올바른 도구를 선택하는 기준은 다음과 같다.

| 상황 | 해결책 | 이유 |
| :--- | :--- | :--- |
| 세션마다 동일한 체크리스트/지식을 주입할 때 | **Skill** | 자동 활성화가 필요한 영구적인 도메인 전문성 |
| 특정 명령어를 명시적으로 실행할 때 | **Slash Command** | 사용자가 직접 제어하는 예측 가능한 액션 (`/rebase`) |
| 주 대화와 분리된 분석이 필요할 때 | **Subagent** | 컨텍스트 오염을 막기 위한 격리된 실행 환경 |

"Claude가 자동으로 적용해야 하는가, 아니면 내가 직접 실행 시점을 결정해야 하는가?" 이 질문이 스킬과 슬래시 커맨드를 구분하는 핵심 기준이다.

#### 3. 파일 시스템 기반의 모듈화 및 범위 관리

스킬은 파일 시스템을 통해 구조화되며, 적용 범위를 명확히 제어할 수 있다.

-   **구조**: `SKILL.md`가 핵심 진입점 역할을 하고, 더 상세한 내용은 `SECURITY_PATTERNS.md` 같은 별도 파일로 분리한 후 상대 경로 링크로 참조할 수 있다. 이는 `SKILL.md`를 500줄 미만으로 간결하게 유지하여 컨텍스트 주입 오버헤드를 줄인다.
-   **범위**: 스킬은 `~/.claude/skills/` (개인 전역) 또는 `.claude/skills/` (프로젝트 한정)에 위치시켜 적용 범위를 제어한다. 프로젝트 레벨 스킬은 Git을 통해 팀원들과 쉽게 공유할 수 있어, 별도 설정 없이 팀 전체의 개발 컨텍스트를 동기화할 수 있다.

### 실전 적용: `ai-study` 위키 기여 가이드라인 리뷰 스킬

#### 1. 스킬 파일 생성
