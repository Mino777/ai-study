## v3.2 — Shared Memories

메모리를 공유 링크로 내보내는 기능. 인증 불필요한 공개 엔드포인트 포함.

```kotlin
data class MemoryShare(
    val shareId: String,       // 12자 랜덤 — 예측 불가능성 확보
    val expiresAt: Instant,    // 만료 시각 (기본값: 24h)
)
```

`GET /api/shared/{shareId}`는 **인증 없이 접근 가능**. 공유 받은 상대가 앱을 설치하지 않아도 메모리 내용을 확인할 수 있어야 하므로. 만료 후 접근하면 404 반환.

3개 엔드포인트:
```
POST   /api/memories/{id}/share    # 공유 링크 생성 (expiresInHours)
GET    /api/shared/{shareId}       # 공유 메모리 조회 (인증 불필요)
DELETE /api/memories/{id}/share    # 공유 해제
```

## v3.3 — Memory Tags

메모리에 사용자 정의 태그를 붙이는 기능. 중간 테이블 패턴.

```
Tag (id, userId, name, color)
  ↕  ← MemoryTag 중간 테이블
Memory (id, ...)
```

**메모리당 최대 5개 제한**을 코드로 강제한다:

```kotlin
// TagService.addTagsToMemory
if (existingTagCount + tagIds.size > 5) {
    throw AidyException(ErrorCode.TAG_LIMIT_EXCEEDED)
}
```

기존 `GET /api/memories`에 `?tag={tagId}` 쿼리 파라미터 추가 — 태그 필터링. 기존 메모리 엔티티 필드는 변경하지 않고 중간 테이블만 추가한다.

## v3.4 — AI Chat Suggestions

채팅 화면 상단에 Claude가 추천 대화 시작 문구를 제시하는 기능. **메모리 컨텍스트를 분석해서 능동적으로 대화를 유도**한다.

### 4가지 추천 타입

| 타입 | 조건 | 예시 |
|------|------|------|
| `followup` | 2주+ 된 미해결 메모리 | "지난번 취업 준비 잘 되어가?" |
| `reminder` | 7일 내 기념일/일정 | "민수 생일 5일 남았어. 뭔가 준비하고 있어?" |
| `exploration` | 2주+ 메모리 없는 카테고리 | "요즘 독서는 안 하는 것 같던데 최근에 읽은 책 있어?" |
| `check_in` | healthScore < 40인 인물 | "지수랑 한동안 연락 없었네. 안부 물어봐?" |

각 추천은 `reason` 필드에 생성 근거를 명시해서 사용자가 왜 이 추천이 나왔는지 알 수 있다.

### 클라이언트 구현

iOS(`ChatSuggestionClient`): 채팅 화면 상단 수평 스크롤 추천 카드. type별 아이콘과 색상, reason 표시. 탭하면 `inputText` 자동 채움 + use API 호출 + 목록에서 제거.

Android(`ChatSuggestionViewModel`): `LazyRow` 추천 카드. 동일 UX.

### markUsed 멱등성

```kotlin
// POST /api/chat/suggestions/{id}/use
// 이미 사용된 추천을 다시 use해도 에러 없음
// 멱등(idempotent): 네트워크 재시도 안전
fun markUsed(userId: Long, suggestionId: Long) {
    val usage = repository.findByUserIdAndSuggestionId(userId, suggestionId)
    if (usage == null) {
        repository.save(ChatSuggestionUsage(userId, suggestionId))
    }
    // else: no-op
}
```

## iOS 반복 크래시 — 4회 (R3)

s29의 가장 큰 장애물. R3에서 iOS 워커가 3회 연속 세션 종료.

### 크래시 패턴

```
[crash-log] ios: UNKNOWN
시스템: Swap 331MB / Claude 6인스턴스 / Memory 17GB
task 표시 완료 → 커밋 전 세션 종료
```

3회 연속 크래시 후 4차 시도에서 성공. **총 4회 시도 = R3에서 iOS가 4개의 Claude 세션을 소비**.

### 원인 분석

crash-log 원인 분류는 UNKNOWN/ERROR였지만, 상황 맥락:
- 6개 Claude 인스턴스 동시 실행 (architect + server + ios + android + ai-study + α)
- iOS WO가 Shared Memories + Memory Tags 2개를 한 번에 처리
- task 완료 표시가 뜨는 시점은 Claude가 모든 파일 편집을 마친 후 → 이후 commit 실행 중 세션 종료 = **토큰 소진 패턴** 유력

### 대응 (다음 스프린트 입력)

```
iOS WO: 2개 기능을 한 번에 → 1개씩 분리해서 전송
프롬프트: 간결하게 (배경 설명 축소, 핵심 작업만)
병렬 인스턴스: 가능하면 5개 이하로 유지
```

## watch-workers 오탐 발견

s29에서 `watch-workers` 명령이 잘못된 신호를 보낸 사례:

- **오탐 상황**: iOS/Android가 아직 dispatch 안 됐는데 idle 감지 → "전원 완료" 보고
- **실제**: dispatch 전 idle 상태를 "작업 완료" idle로 잘못 판정

```bash
# watch-workers 현재 감지 로직 (버그)
if pane_idle: report "완료"

# 필요한 로직
if was_dispatched AND now_idle: report "완료"
# else: report "대기 중 (미dispatch)"
```

dispatch 여부를 추적하는 상태가 없어서 발생. 다음 바선생 업그레이드에서 "dispatch 후 최소 1분 working 확인" 로직 추가 예정.

## AI Chat Suggestions 패턴의 의미

단순 채팅 인터페이스에서 **맥락 인식 대화 유도**로 전환하는 분기점.

기존 aidy 채팅 흐름:
```
사용자가 먼저 말을 걸어야 대화 시작
```

v3.4 이후:
```
Claude가 메모리/기념일/건강점수를 분석
→ "지금 이 대화를 시작하면 의미 있겠다" 판단
→ 채팅 화면 상단에 추천 표시
→ 사용자는 탭 한 번으로 맥락 있는 대화 시작
```

이는 기억 보조 앱에서 **능동적 관계 관리 코치**로의 전환이다. 사용자가 기억하지 못하는 것을 Claude가 대신 기억하고 때를 알려준다.
