2. **debate 9 에이전트 응답 구조 복잡도** (types.ts 조사):
   - 중첩 배열: `ResearchDebateResult.rounds[]`, `RiskDebateResult.messages[]`

   - 독립 평면: `TraderProposal` (가장 단순)
   - 이 복잡도는 Journal 012/013 (4 analyst 평면 구조)보다 한 단계 높다

   - 모두 type guard 기반
   - `checkAgentReportBase` 공유로 base 검증 일관성 유지
   - 새 패턴 도입 시 *두 패턴 공존* 기간이 발생 (Zod + type guard 혼재)

## TraderProposal validator 설계

```typescript
export function validateTraderProposal(value: unknown): ValidationResult<TraderProposal> {
  if (!isRecord(value)) return { ok: false, issues: ['root: object required'] };

  const issues: string[] = [];

  if (typeof value.action !== 'string' || !SIGNAL_VALUES.includes(value.action as ...)) {
    issues.push(`action: must be one of ${SIGNAL_VALUES.join('|')}`);
  }
  if (typeof value.position_size_pct !== 'number' || value.position_size_pct < 0 || value.position_size_pct > 100) {
    issues.push('position_size_pct: number 0-100 required');
  }
  if (typeof value.entry_strategy !== 'string' || value.entry_strategy.length === 0) issues.push(...);
  if (typeof value.exit_strategy !== 'string' || value.exit_strategy.length === 0) issues.push(...);
  if (typeof value.time_horizon !== 'string' || !['단기', '중기', '장기'].includes(value.time_horizon)) {
    issues.push('time_horizon: must be 단기|중기|장기');
  }
  if (typeof value.reasoning !== 'string' || value.reasoning.length === 0) issues.push(...);

  if (issues.length > 0) return { ok: false, issues };
  return { ok: true, value: value as unknown as TraderProposal };
}
```

**AgentReport 확장 안 함** → `checkAgentReportBase` 재사용 불가 → 독립 체크. 이게 debate 에이전트의 *공통 특성*일지, 아니면 Trader만의 특이점일지 Journal 016에서 확인.
