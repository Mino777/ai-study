### WO 없는 직접 dispatch (s18)

s18에서 WO 발행 없이 스펙만 정의하고 바로 dispatch. 라운드를 빠르게 돌릴 수 있지만 done 폴더에 기록이 없다.

```
WO 발행 포함 흐름:
  스펙 정의 → WO 파일 생성 → in-progress/ → 워커 dispatch → done/ 이동
  
WO 없는 직접 dispatch (s18):
  스펙 정의 → 워커 직접 dispatch
  (done 폴더 비어있음 → 추적 불가)
```

tradeoff: 단기 속도 vs 장기 추적성. s18 retro에서 "속도 vs 추적성 트레이드오프를 의식적으로 결정하지 않았다"고 명시.

### dispatch 체이닝 금지 규칙 발견

s19-R5에서 iOS dispatch 누락 사고:

```bash
# 위험한 패턴 — iOS dispatch가 조용히 실패
./architect-cli.sh send server "[R5] 작업 지시..." && \
./architect-cli.sh send ios "[R5] 작업 지시..."
# && 체이닝: 서버 성공해도 iOS 전달 실패 시 에러 없이 통과

# 안전한 패턴 — 각각 별도 호출
./architect-cli.sh send server "[R5] 작업 지시..."
./architect-cli.sh send ios "[R5] 작업 지시..."
```

iOS R8 빌드 28분 병목도 발생. DerivedData 잠금으로 재빌드. 빌드 전 `rm -rf ~/Library/Developer/Xcode/DerivedData/<project>` 선제 실행 필요.

MVP(v1.0) 달성 직후 기능 동결하고 품질만 집중.

| 라운드 | 작업 내용 |

| R2 | Android Insights UI (iOS 패리티) |
