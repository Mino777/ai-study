## 갭 / 맥락

2026년 4월, AI 코딩 도구들이 성숙하면서 새로운 병목이 드러났다:
**코드는 AI가 짜주는데, 뭘 만들어야 하는지를 전달하는 게 여전히 수동이다.**

기존 워크플로:
```
아이디어 → Figma에서 디자인 (수동, 시간 소요)
  → Zeplin/PDF로 스펙 전달 (정보 손실)
    → 개발자가 해석해서 구현 (오해 발생)
```

Claude Design이 이 간극을 메운다:
```
아이디어 → Claude Design에서 프로토타입 (대화 기반, 분 단위)
  → 핸드오프 번들로 Claude Code에 전달 (구조화된 데이터, 손실 없음)
    → Claude Code가 바로 구현 시작
```

## Claude Design 핵심 개요

### 무엇인가

- **출시일**: 2026년 4월 17일 (Anthropic Labs)
- **엔진**: Claude Opus 4.7 (해상도 1,568px → 2,576px, 1.15MP → 3.75MP, **3.26x 향상**)
- **성격**: AI 이미지 생성기가 **아님**. 프로토타입, 슬라이드, UI 목업, 마케팅 에셋을 만드는 **협업 비주얼 도구**
- **대상**: 디자이너가 아닌 사람 (PM, 파운더, 마케터, 엔지니어)이 아이디어를 비주얼로 빠르게 전환

### UI 구조

```
┌─────────────────┬──────────────────────┐
│                 │                      │
│   Chat Panel    │   Design Canvas      │
│   (대화/지시)    │   (실시간 렌더링)      │
│                 │                      │
│   "로그인 화면   │   ┌──────────────┐   │

```

## 핵심 기능 5가지

### 1. 디자인 시스템 자동 구축

온보딩 시 코드베이스 + 디자인 파일을 읽어서 팀의 디자인 시스템을 자동 생성:
- 색상 팔레트, 타이포그래피, 컴포넌트
- 이후 모든 프로젝트에 자동 적용
- 팀별 여러 디자인 시스템 유지 가능
- 시간이 지나면서 계속 다듬기 가능

**Aidy 적용 가능성**: DESIGN.md를 Claude Design에 넘기면 자동으로 Aidy 브랜드 시스템 구축 → 모든 UI 목업이 일관된 스타일.

### 2. 다양한 입력 방식

| 입력 | 설명 |
|------|------|
| 텍스트 프롬프트 | "대시보드 화면 만들어줘" |
| 이미지 업로드 | 스케치, 스크린샷, 참고 디자인 |
| 문서 업로드 | DOCX, PPTX, XLSX |
| 코드베이스 | 기존 프로젝트 구조 참조 |
| 웹 캡처 | 실제 웹사이트 요소를 캡처해서 프로토타입에 반영 |

### 3. 정밀 조정 도구

- **인라인 코멘트**: 특정 요소에 직접 피드백
- **직접 텍스트 편집**: 캔버스에서 바로 수정
- **커스텀 슬라이더**: 간격, 색상, 레이아웃을 실시간 조절
- **대화 기반 반복**: 멀티턴 대화로 계속 다듬기

### 4. 내보내기 옵션

| 포맷 | 용도 |
|------|------|
| **Canva** | 팀 협업 편집 (런치 파트너) |
| **PPTX** | 프레젠테이션 |
| **PDF** | 인쇄/공유 |
| **HTML** | 빠른 배포, 개발자 핸드오프 |
| **공유 URL** | 팀 내 링크 공유 (보기/편집 권한) |
| **Claude Code 핸드오프** | **구조화된 번들로 바로 구현** |

### 5. Claude Code 핸드오프 (가장 중요)

```
Claude Design → Export → Copy handoff command
  → Paste into Claude Code → Claude Code fetches design bundle
    → CSS, 컴포넌트 구조, 레이아웃 의도 포함
      → 바로 빌드 시작
```

**이것이 PDF 스펙이나 Zeplin 링크와 근본적으로 다른 이유**:
- 구조화된 데이터 (CSS, 컴포넌트 트리, 레이아웃)
- 정보 손실 없음 (사람의 해석 불필요)
- zip 파일로도 내보내기 가능 (Claude Code 외 다른 코딩 에이전트에도 전달 가능)

## 포지셔닝 — Figma vs Canva vs Claude Design

```
Claude Design          Figma              Canva
(아이디어 → 프로토타입)  (디자인 정밀 작업)   (퍼블리싱)
     │                    │                  │
     ▼                    ▼                  ▼
  탐색/발산             수렴/완성           배포/공유
  "10개 방향을           "1px까지            "팀에 배포"
   분 단위로 탐색"       다듬기"
```

- **Claude Design ≠ Figma 대체**: 탐색 단계에서 빠르게 방향 잡기
- **Claude Design + Figma**: Claude Design으로 10개 방향 탐색 → Figma에서 선택한 방향 정밀 작업
- Figma 주가 7% 하락했지만, Anthropic은 "보완재"로 포지셔닝

### 가격

- Pro ($20/mo), Max ($100-200/mo), Team ($25-30/seat/mo), Enterprise — **추가 비용 없음**
- 채팅/Claude Code와 별도의 주간 사용량 제한
- 무료 티어에서는 사용 불가

## AI Agent Directive

### Trigger
- UI/UX 프로토타입이 필요할 때
- 디자인 시안 없이 기능을 구현해야 할 때
- 프레젠테이션/피치덱 제작 시
- 기존 디자인을 코드로 빠르게 전환할 때

### Prerequisites
- Claude Pro/Max/Team/Enterprise 구독
- 프로젝트의 디자인 시스템 (DESIGN.md 또는 기존 코드베이스)

### Actionable Steps

1. **디자인 시스템 먼저 세팅** — 온보딩에서 코드베이스를 읽게 해서 브랜드 일관성 확보. DESIGN.md가 있으면 업로드.

2. **프로토타입은 텍스트 프롬프트로 시작** — 완벽한 프롬프트를 쓸 필요 없음. "대시보드 화면 만들어줘" 수준으로 시작 → 대화로 다듬기.

3. **웹 캡처로 현실감 확보** — 기존 앱의 UI를 캡처해서 프로토타입에 반영하면 실제 제품과 유사한 목업 가능.

4. **Claude Code 핸드오프 활용** — 디자인 확정 후 핸드오프 번들로 내보내기 → Claude Code에서 바로 구현. 수동 스펙 작성 불필요.

5. **Aidy 워크플로 적용 가능성**:
   ```
   기존: Architect가 텍스트로 UI 위치 설명 → 워커가 해석
   개선: Claude Design으로 목업 생성 → 핸드오프 번들을 WO에 첨부
         → 워커가 시각적 참조 + 구조 데이터로 정확한 구현
   ```

### Anti-patterns

- **이미지 생성 도구로 착각**: Midjourney/DALL-E와 다름. 일러스트/사진 생성이 아닌 UI/프로토타입 도구.
- **Figma 완전 대체로 사용**: 정밀 디자인 작업은 여전히 Figma가 우위. 탐색 단계에서만 활용.
- **핸드오프 없이 스크린샷만 전달**: Claude Design의 가치는 구조화된 핸드오프. 스크린샷 전달은 기존과 동일한 정보 손실.

## 프론티어 디자인: 코드 기반 인터랙티브 경험

정적 시각물을 넘어 **코드 기반 프로토타입**도 생성 가능:

| 카테고리 | 예시 |
|---------|------|
| 3D 시각화 | 인터랙티브 3D 지구본, 데이터 시각화 |
| 음성 인터페이스 | 음성 기반 프로토타입 |
| 비디오/모션 | 인터랙티브 효과, 셰이더 |
| AI 내장 | 프로토타입 자체에 AI 기능 포함 |

현재 초기 단계로 거친 부분이 있으나, 디자인 도구의 가능성을 코드 실행 영역까지 확장한 점이 주목할 만하다.

## AI 엔지니어링 관점 시사점

1. **Context Engineering 실전**: 디자인 시스템 온보딩이 곧 컨텍스트 엔지니어링. GitHub 코드 + Figma + 스타일 노트를 읽어 모델에 팀의 디자인 언어를 주입
2. **에이전트 파이프라인 완성**: Design → Code 핸드오프는 에이전트 간 구조화된 데이터 전달의 실전 사례. 번들 포맷이 에이전트 간 통신 프로토콜 역할
3. **멀티모달 입력의 가치**: 텍스트 + 이미지 + 문서 + 웹 캡처 — 단일 모달보다 풍부한 컨텍스트가 더 정확한 결과를 만듦
4. **해상도가 곧 능력**: Opus 4.7의 3.26x 해상도 향상이 디자인 도구를 가능하게 함. 모델 능력이 제품 카테고리를 결정하는 사례

## Aidy 프로젝트 적용 계획

### Phase 1: 디자인 시스템 세팅
- DESIGN.md + 현재 iOS/Android 코드베이스를 Claude Design에 넘겨서 Aidy 디자인 시스템 구축

### Phase 2: WO에 프로토타입 첨부
- 신규 기능 WO 발행 시 Claude Design으로 목업 생성
- 핸드오프 번들 또는 스크린샷을 WO에 첨부
- 워커가 "어떤 탭, 어느 화면에 배치"를 시각적으로 확인 (s22 retro 교훈 해결)

### Phase 3: Claude Code 핸드오프 파이프라인
- Claude Design → 핸드오프 → Claude Code(워커 세션)에 전달
- 스펙 텍스트 + 시각 자료 + 구조 데이터 삼중 전달
