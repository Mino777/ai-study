### Push Notification — 인프라만

실제 FCM/APNs 발송은 구현하지 않았다. 토큰 저장 API만:

```
POST /api/notifications/token    ← 토큰 등록
DELETE /api/notifications/token  ← 토큰 해제
```

retro 자기 점검: "발송 없는 토큰 저장은 dead code가 될 수 있다." 이후 스프린트에서 FCM 서버 키 설정 + 발송 로직 추가 예정으로 미뤘지만, 실제로는 s27까지 구현 미완.

### Gate-1 또 생략 (4번째)

retro: *"Gate-1 또 생략. s20 retro에서 'R5, R10에서 gate-reviewer 실행'을 다짐했으나 이번에도 한 번도 실행 안 함. 같은 실수 반복 — 우선순위 판단이 아니라 **귀찮아서** 건너뜀."*

## s23 — Gate-1 효과 증명 + ADR-011

### 구현 내용

| 라운드 | 기능 |
|--------|------|
| R1~R4 | Chat Bookmarks (롱프레스 토글) + AI Feedback (good/bad) — v1.4, Gate-1 PASS |
| R5~R8 | Chat Topics (AI 클러스터링) + Chat Export (text/json) — v1.5, Gate-1 FAIL→수정→PASS |
| R9 | 에지 케이스 테스트 보강 (+22+33건) |

### Gate-1이 잡아낸 스키마 이슈

Chat Topics API의 응답 스키마 검증 중 발견:

```json
// 스펙 (api-contract.md §4.21)
{
  "firstMessageAt": "2026-04-10T09:00:00Z",  // 첫 메시지
  "lastMessageAt": "2026-04-17T21:30:00Z"     // 마지막 메시지
}

// 실제 구현 (서버 워커 코드)
{
  "firstMessageAt": "2026-04-17T21:30:00Z",   // ← 동일값
  "lastMessageAt": "2026-04-17T21:30:00Z"     // ← 그룹 내 최신값만 반환
}
```

빌드 통과, 단위 테스트 통과(동일값 검증 안 함), **Gate-1에서만 발견**. R8에서 즉시 수정 후 PASS.

### ADR-011 — AgentCompiler 불채택

허브 dispatch 이슈 처리 중 AgentCompiler(동적 워커 라우팅) 아이디어가 제안됐지만 불채택:

```
ADR-011: WO 라우팅 정적 유지, AgentCompiler 불채택
이유: 워커 3개 고정 → 정적 dispatch로 충분 (YAGNI)
      동적 라우팅 필요 시점: 워커 수가 변동될 때 재검토
```
