## 패턴: UI Test First, Identify Second

이번 스프린트에서 발견한 순서:

1. **accessibility identifier 전수 부착** — 테스트 대상을 먼저 식별 가능하게 만든다
2. **UI 테스트 작성** — identifier 기반으로 화면 인터랙션 자동화
3. **QA 에이전트 연동** — 한 명령으로 실행 + 결과 수집

이 순서가 뒤집히면(테스트 먼저 → identifier 나중) Gate에서 identifier mismatch가 연쇄 발생한다. s9 R3의 chatRetryButton 인시던트가 그 증거.
