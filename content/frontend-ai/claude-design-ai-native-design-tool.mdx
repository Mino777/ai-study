## 요약

2026년 4월 17일, Anthropic이 **Claude Design**을 출시했다. Claude Opus 4.7의 향상된 비전 능력(해상도 1,568px → 2,576px) 위에 구축된 AI 네이티브 디자인 도구로, 텍스트 프롬프트에서 폴리시된 프로토타입까지, 그리고 Claude Code 핸드오프를 통해 프로덕션 코드까지 이어지는 **탐색 → 프로토타입 → 코드** 파이프라인을 제공한다.

## 핵심 아키텍처

```
[입력]                    [Claude Design]              [출력]
텍스트 프롬프트 ──┐
참고 이미지 ──────┤
문서 (DOCX/PPTX) ┼──→ Opus 4.7 비전 ──→ 디자인 ──→ PDF / PPTX
코드베이스 ───────┤     + 디자인 시스템       │       Canva 내보내기
웹 캡처 ──────────┘                          │       독립형 HTML
                                             ↓       Claude Code 핸드오프
                                    대화형 정제 루프      ↓
                                    (채팅/주석/편집/슬라이더)  프로덕션 코드
```

## 4단계 워크플로우

### 1. 디자인 시스템 온보딩

Claude가 팀의 기존 자산을 읽고 **디자인 시스템을 자동 구축**한다:

| 입력 소스 | 역할 |
|-----------|------|
| GitHub 저장소 | 컬러 팔레트, 타이포그래피, 컴포넌트 패턴 추출 |
| Figma 파일 | 시각적 스타일 가이드 분석 |
| 폰트/로고 폴더 | 브랜드 아이덴티티 반영 |
| 스타일 노트 | 팀의 디자인 원칙 반영 |

이후 **모든 프로젝트가 이 시스템을 자동 상속**. 팀은 여러 디자인 시스템을 유지하고 프로젝트별로 전환 가능.

### 2. 프로젝트 시작

다중 입력 지원:
- **텍스트 프롬프트**: "대시보드 페이지를 만들어줘"
- **이미지 업로드**: 참고 디자인, 스케치, 스크린샷
- **문서 변환**: DOCX, PPTX, XLSX → 시각적 디자인으로 변환
- **웹 캡처**: 실제 웹사이트에서 요소를 직접 추출 → 프로토타입이 실제 제품처럼 보임

### 3. 대화형 정제

4가지 정제 인터페이스:
- **채팅**: 자연어로 변경 요청 ("헤더를 더 크게")
- **인라인 주석**: 특정 요소에 직접 피드백
- **직접 편집**: 텍스트, 레이아웃 등을 직접 수정
- **커스텀 슬라이더**: 스타일 파라미터를 세밀하게 조정

### 4. 내보내기 / 핸드오프

| 용도 | 포맷 |
|------|------|
| 마케팅/발표 | PPTX, PDF |
| 협업 | Canva (완전 편집 가능) |
| 개발 | Claude Code 핸드오프 번들 |
| 공유 | 읽기 전용 URL, 독립형 HTML |

## Claude Code 핸드오프 파이프라인

AI 엔지니어링 관점에서 가장 의미있는 기능. **디자인→코드 파이프라인이 Anthropic 생태계 내에서 완결**된다.

```
Claude Design                          Claude Code
  ┌─────────────┐    핸드오프 번들     ┌─────────────┐
  │ 디자인 완성  │ ──────────────────→ │ 번들 수신    │
  │ (시각 자산   │    단일 지시로      │ 코드 생성    │
  │  + 스펙)     │    전달             │ 프로덕션     │
  └─────────────┘                     └─────────────┘
```

핸드오프 번들에는 디자인 시스템 정보, 컴포넌트 구조, 스타일 사양이 포함되어 Claude Code가 맥락을 완전히 이해한 상태에서 코드를 생성한다.

**의미**: 기존에는 디자인(Figma) → 개발자 해석 → 코드로 3단계 핸드오프가 필요했다. Claude Design은 이를 디자인 → 자동 핸드오프 → 코드로 축소. 중간 해석 손실이 없다.

## 프론티어 디자인

정적 시각물을 넘어 **코드 기반 인터랙티브 경험**도 생성 가능:

- **3D 시각화**: 인터랙티브 3D 지구본, 데이터 시각화
- **음성 인터페이스**: 음성 기반 프로토타입
- **비디오/모션**: 인터랙티브 효과, 셰이더
- **AI 내장**: 프로토타입 자체에 AI 기능 포함

현재 초기 단계로 거친 부분이 있으나, 디자인 도구의 가능성 범위를 코드 실행 영역까지 확장한 점이 주목할 만하다.

## 기술 기반: Opus 4.7 비전 업그레이드

| 항목 | Opus 4.6 | Opus 4.7 |
|------|----------|----------|
| 비전 해상도 | 1,568px (1.15MP) | 2,576px (3.75MP) |
| 해상도 배수 | 기준 | **3.26x** |

이 해상도 향상이 Claude Design의 디자인 분석 품질을 직접 가능하게 한다. 세밀한 UI 요소, 타이포그래피, 스페이싱을 정확하게 인식.

## 시장 포지셔닝

| | Claude Design | Figma | Canva |
|--|---------------|-------|-------|
| **타겟** | 비디자이너 (창업가, PM) | 전문 디자이너 | 일반 사용자 |
| **시작점** | 텍스트 프롬프트 / 아이디어 | 빈 캔버스 | 템플릿 |
| **강점** | 아이디어→시각물 속도 | 정밀 제어, 협업 | 접근성, 템플릿 |
| **코드 연결** | Claude Code 직접 핸드오프 | DevMode (수동 해석) | 없음 |

Anthropic은 Figma/Canva의 **보완재**로 포지셔닝. "디자인 도구에서 시작하지 않는 사람들"을 위한 도구.

## 실제 성과 데이터

| 기업 | Before | After |
|------|--------|-------|
| **Brilliant** | 복잡한 페이지에 20+ 프롬프트 필요 (다른 도구) | 2개 프롬프트로 완성 |
| **Datadog** | 반복 작업에 1주일 | 단일 대화로 단축 |

## 가용성

- **대상**: Claude Pro, Max, Team, Enterprise 구독자
- **비용**: 구독 한도 내 무료, 초과 시 추가 비용
- **Enterprise**: 기본 비활성화, 관리자가 조직 설정에서 활성화 필요
- **상태**: Research Preview (Anthropic Labs)

## AI 엔지니어링 관점 시사점

1. **Context Engineering 실전**: 디자인 시스템 온보딩이 곧 컨텍스트 엔지니어링. GitHub 코드 + Figma + 스타일 노트를 읽어 모델에 팀의 디자인 언어를 주입
2. **에이전트 파이프라인 완성**: Design → Code 핸드오프는 에이전트 간 구조화된 데이터 전달의 실전 사례. 번들 포맷이 에이전트 간 통신 프로토콜 역할
3. **멀티모달 입력의 가치**: 텍스트 + 이미지 + 문서 + 웹 캡처 — 단일 모달보다 풍부한 컨텍스트가 더 정확한 결과를 만듦
4. **해상도가 곧 능력**: Opus 4.7의 3.26x 해상도 향상이 디자인 도구를 가능하게 함. 모델 능력이 제품 카테고리를 결정하는 사례
