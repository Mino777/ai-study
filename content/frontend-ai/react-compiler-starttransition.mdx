## 갭 / 맥락

Next.js 16에서 `reactCompiler: true`를 활성화하면, `useEffect` 내의 `setState` 호출에 `react-hooks/set-state-in-effect` 린트 에러가 발생한다.

```typescript
// ❌ React Compiler 경고
useEffect(() => {
  setLoaded(true);  // Compiler: "cascading render risk"
}, []);
```

이 패턴은 hydration guard(서버/클라이언트 불일치 방지)에서 매우 흔하다. tarosaju에서 7건 발생.

---

## 핵심 패턴

```typescript
import { startTransition } from "react";

// ✅ React Compiler 통과
useEffect(() => {
  startTransition(() => setLoaded(true));
}, []);
```

### 적용 대상 유형

| 유형 | 예시 | 왜 useEffect + setState가 필요한가 |
|------|------|-----------------------------------|
| Hydration guard | `setLoaded(true)` | 서버/클라이언트 렌더 불일치 방지 |
| Profile auto-fill | `setName(user.name)` | 클라이언트 로컬스토리지에서 읽기 |
| Client capability | `setSupportsTouch(true)` | window/navigator 접근은 클라이언트만 |
| Cleanup state | `setCleanup(false)` | 컴포넌트 언마운트 전 상태 정리 |

### Before / After

```typescript
// Before (7건 lint 에러)
useEffect(() => {
  setLoaded(true);
  setShow(true);
  setName(savedName);
  setYear(savedYear);
}, []);

// After (0건 lint 에러)
useEffect(() => {
  startTransition(() => {
    setLoaded(true);
    setShow(true);
    setName(savedName);
    setYear(savedYear);
  });
}, []);
```

## 주의사항

- `startTransition` 내의 업데이트는 **non-urgent**로 처리되므로, 즉각적인 UI 반응이 필요한 경우(입력 필드 등)에는 부적합
- Hydration guard, 초기 로드 상태 같은 "보여지기 전 설정"에만 사용
- `next.config.ts`에서 `reactCompiler: true`가 활성화된 경우에만 해당

## AI Agent Directive

### Trigger
- React Compiler 활성화 후 `react-hooks/set-state-in-effect` 경고 발생 시
- Next.js 16+ 프로젝트에서 useEffect 내 setState 패턴이 있을 때

### Actionable Steps
1. `useEffect` 내 모든 `setState` 호출을 `startTransition(() => setState(...))` 로 감싸기
2. 즉각 반응이 필요한 setState는 제외 (입력 핸들러 등)
3. `import { startTransition } from "react"` 추가

### Anti-patterns
- **Compiler 비활성화로 우회** — 최적화 효과를 포기하게 됨
- **eslint-disable로 무시** — 실제 cascading render 이슈를 가릴 수 있음
