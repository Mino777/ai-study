## Swift의 DynamicContext 구조체 설계

moneyflow-ios에서 사용하는 Context 타입:

```swift
struct DynamicContext: Codable {
  let cartItems: [CartItem]
  let userProfile: UserProfile
  let liveInventory: [String: Int]  // productId → stock
  let timestamp: String
  let deviceInfo: DeviceInfo
}
```

## ViewModel에서 Context 수집

```swift
@MainActor
class ProductRecommendationViewModel: ObservableObject {
  @Published var context: DynamicContext?
  
  func collectContext() async {
    async let cart = cartService.fetchCart()
    async let user = userService.currentUser()
    async let inventory = inventoryService.fetchLiveStocks()
    
    let (cartItems, userProfile, stocks) = await (cart, user, inventory)
    
    self.context = DynamicContext(
      cartItems: cartItems,
      userProfile: userProfile,
      liveInventory: stocks,
      timestamp: ISO8601DateFormatter().string(from: Date()),
      deviceInfo: DeviceInfo.current
    )
  }
}
```

## LLM에 injection할 때 JSON 포맷

```swift
let jsonEncoder = JSONEncoder()
jsonEncoder.outputFormatting = [.prettyPrinted, .sortedKeys]
let contextJSON = try jsonEncoder.encode(context)

let prompt = """

```

## 성능 최적화: 병렬 로딩

3개의 API 호출을 순차적으로 하면 총 시간 = A + B + C (가정: 각 300ms)

async let을 사용하면 병렬화 → 최악의 경우 max(A, B, C) ≈ 300ms

moneyflow-ios의 측정: 800ms → 280ms로 개선
