## Context Filtering의 중요성

수집한 context를 **그대로** LLM에 보내면:
- 평균 토큰 사용 200 → 350 (75% 증가)
- 불필요한 노이즈로 인해 hallucination 증가

## Swift 구현: FilteredContext

```swift
extension DynamicContext {
  func filtered(for intent: UserIntent) -> FilteredContext {
    var items = self.cartItems
    
    // 추천 요청이면 최근 본 상품만, 장바구니 역할은 약하게
    if intent == .getRecommendation {
      items = items.filter { $0.viewedAt > Date().addingTimeInterval(-3600) }
    }
    
    // 배송 정보 제거 (추천에는 불필요)
    return FilteredContext(
      cartCount: items.count,
      lastViewed: items.first?.productId,
      userTier: userProfile.tier,  // 할인 정보만
      timestamp: timestamp
    )
  }
}
```

## 결과: moneyflow-ios의 측정

| 시나리오 | 토큰 (필터링 전) | 토큰 (필터링 후) | 정확도 |
|---------|----------|-----------|--------|
| 상품 추천 | 320 | 190 | +8% |
| 장바구니 요약 | 280 | 230 | +2% |
| 체크아웃 도움말 | 450 | 310 | +12% |

## 주의점

**언제 filtering을 할 것인가?**
- ✅ LLM 호출 직전 (비용 절감)
- ❌ context 수집 시점에 (나중에 필요할 수 있음)

moneyflow-ios는 DynamicContext를 캐시했다가, 각 intent별로 다르게 filtered하는 구조를 사용한다.
