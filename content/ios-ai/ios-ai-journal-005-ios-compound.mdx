## iOS AI Journal 005: iOS Compound - 배포 후 지식 축적

iOS 배포는 웹과 다르다. **앱스토어 심사 → 배포 → 사용자 피드백** 사이클이 최소 며칠이다. 그 사이에 발생하는 지식을 어떻게 다음 스프린트에 넣을까?

### iOS 특화 Compound 사이클

```
코드 작성 → PR → 앱스토어 심사 → 배포
                                    ↓
                          사용자 피드백 / 크래시 리포트
                                    ↓
                         /ios-investigate → /ios-bugfix
                                    ↓
                          docs/solutions/ios/ 기록
                                    ↓
                          다음 스프린트 NEXT.md 입력 ←
```

### docs/solutions/ios/ 카테고리

```
docs/solutions/ios/
├── crashes/          ← 크래시 해결 기록
├── rib-patterns/     ← RIBs 관련 패턴
├── appstore/         ← 심사 관련 이슈
├── performance/      ← 성능 최적화
├── migration/        ← Swift/Xcode 버전 마이그레이션
└── deferred/         ← 미해결 / 추후 재시도 이슈
```

### 회고 자동화: /ios-compound

```markdown
# .claude/commands/ios-compound.md

## iOS Compound 단계

### 1. 이번 스프린트 결과 정리
- 해결한 버그 목록
- 추가한 기능 목록
- 도입한 패턴/도구

### 2. docs/solutions/ios/ 기록
각 해결책을 카테고리에 맞게 저장:
- 문제 상황
- 원인
- 해결책
- 재발 방지 방법

### 3. deferred 업데이트
미해결 이슈 → docs/solutions/ios/deferred/ 에 기록
(시도한 것, 실패 이유 포함)

### 4. NEXT.md 업데이트
다음 스프린트 후보 3개 작성
(이번 회고에서 발견된 것 우선)

### 5. CHANGELOG 업데이트
앱스토어 출시 노트 초안 작성
```

### 앱스토어 심사 실패도 자산

```markdown
# docs/solutions/ios/appstore/2026-04-10-review-rejection-guideline-4-3.md

## 심사 거절: Guideline 4.3 (Spam)

**상황**: 타로 앱 v3.2 심사 거절
**이유**: 기존 앱과 기능이 유사하다는 판정
**해결책**: 차별화 기능 스크린샷 추가 + 심사 메모 작성
**재심사 결과**: 통과
**다음 버전 시**: 처음부터 차별화 포인트를 스크린샷에 넣기
```

이처럼 **심사 거절 이유도 축적**하면, 다음 버전에서 같은 이유로 거절당하는 일이 없다.
