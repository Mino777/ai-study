## iOS AI Journal 001: Swift 코드 규칙 자동 강제

AI가 Swift 코드를 쓸 때 SwiftLint 규칙을 모른다고 가정하는 게 현실적이다. 매번 "SwiftLint 규칙 지켜라"를 프롬프트에 넣는 대신, **구조적으로 강제**한다.

### PreToolUse 훅으로 Write 차단

Claude Code의 `PreToolUse` 훅을 활용한다. Swift 파일을 쓰기 전에 자동으로 lint 검사를 실행.

```json
// .claude/settings.json
{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "Write",
        "hooks": [
          {
            "type": "command",
            "command": "bash .claude/hooks/swift-pre-write.sh"
          }
        ]
      }
    ]
  }
}
```

```bash
# .claude/hooks/swift-pre-write.sh
#!/bin/bash

# Write하려는 파일이 Swift 파일인지 확인
FILE_PATH="${CLAUDE_TOOL_INPUT_FILE_PATH}"

if [[ "$FILE_PATH" == *.swift ]]; then
  # 임시 파일로 저장 후 lint 검사
  echo "${CLAUDE_TOOL_INPUT_CONTENT}" > /tmp/claude_check.swift
  
  if ! swiftlint lint --path /tmp/claude_check.swift --quiet 2>/dev/null; then
    echo "SwiftLint 규칙 위반 감지. 코드를 수정하세요."
    exit 1  # 훅 실패 → AI가 재시도
  fi
fi
```

### PostToolUse 훅으로 포맷 자동 적용

쓴 후에는 SwiftFormat을 자동으로 적용한다:

```json
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Write",
        "hooks": [
          {
            "type": "command",
            "command": "bash .claude/hooks/swift-post-write.sh"
          }
        ]
      }
    ]
  }
}
```

```bash
# .claude/hooks/swift-post-write.sh
#!/bin/bash

FILE_PATH="${CLAUDE_TOOL_OUTPUT_FILE_PATH}"

if [[ "$FILE_PATH" == *.swift ]]; then
  swiftformat "$FILE_PATH" 2>/dev/null
  echo "SwiftFormat 적용 완료: $FILE_PATH"
fi
```

### 실제 효과

**훅 없을 때**: AI가 쓴 코드 → 개발자가 수동으로 lint 돌림 → 수정 → 재커밋  
**훅 있을 때**: AI가 쓰기 시도 → 자동 lint → 위반시 자동 재시도 → 통과된 코드만 저장

**반복 수정 횟수: 평균 3회 → 0회**

### 주의사항

- SwiftLint, SwiftFormat이 로컬에 설치되어 있어야 함 (`brew install swiftlint swiftformat`)
- CI에도 동일 버전 설치 필요 (로컬과 CI 버전 불일치 주의)
- 너무 엄격한 룰은 AI가 루프에 빠질 수 있음 → warning은 허용, error만 차단 권장
