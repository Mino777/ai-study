## 한 줄 요약

"SwiftLint 규칙 지켜라"를 매번 프롬프트에 쓰지 않는다. PreToolUse 훅이 위반 코드를 쓰기 전에 차단하고, PostToolUse 훅이 SwiftFormat을 자동 적용한다.

---

## 문제

Claude가 Swift 코드를 생성할 때 SwiftLint 규칙을 모른다고 가정하는 게 현실적이다. 프로젝트마다 다른 `.swiftlint.yml` 규칙이 있고, AI는 이걸 정확히 지키지 않는다.

**기존 워크플로우**:
```
AI가 코드 생성 → 개발자가 swiftlint 돌림 → 위반 확인 → 수동 수정 → 재커밋
```

반복 수정 평균 3회. 개발자가 lint 결과를 다시 AI에게 붙여넣어야 함.

---

## 실제 SwiftLint / SwiftFormat 규칙 구성

프로젝트의 `.swiftlint.yml`:

```yaml
disabled_rules:
  - trailing_whitespace
  - line_length

opt_in_rules:
  - empty_count
  - force_unwrapping        # ! 사용 금지 (opt-in으로 에러 처리)
  - implicitly_unwrapped_optional

excluded:
  - Pods
  - .build
  - Generated

force_unwrapping:
  severity: error           # warning이 아닌 error로 차단
```

`.swiftformat` 파일:
```
--indent 4
--swiftversion 5.9
--disable redundantReturn
--disable trailingCommas
--maxwidth 120
```

## PreToolUse 훅: 쓰기 전에 차단

Claude Code의 PreToolUse 훅을 활용한다. Swift 파일 Write 직전 자동으로 lint 검사 실행.

```json
// .claude/settings.json
{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "Write",
        "hooks": [
          {
            "type": "command",
            "command": "bash .claude/hooks/swift-pre-write.sh"
          }
        ]
      }
    ]
  }
}
```

```bash
#!/bin/bash
# .claude/hooks/swift-pre-write.sh

FILE_PATH="${CLAUDE_TOOL_INPUT_FILE_PATH}"

if [[ "$FILE_PATH" != *.swift ]]; then
  exit 0
fi

# 작성 예정 콘텐츠를 임시 파일로 저장 후 lint 검사
echo "${CLAUDE_TOOL_INPUT_CONTENT}" > /tmp/claude_swift_check.swift

# error 레벨만 차단 (warning은 허용)
if swiftlint lint --path /tmp/claude_swift_check.swift --quiet 2>/dev/null | grep -q "error:"; then
  echo "SwiftLint error 위반 감지. 코드를 수정 후 다시 시도하세요."
  cat /tmp/claude_swift_check.swift | swiftlint lint --path /tmp/claude_swift_check.swift 2>/dev/null | grep "error:"
  exit 1
fi

rm -f /tmp/claude_swift_check.swift
exit 0
```

훅이 `exit 1`을 반환하면 Claude는 위반 내용을 보고 코드를 수정 후 재시도한다.

## PostToolUse 훅: 쓴 후 자동 포맷

```json
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Write",
        "hooks": [
          {
            "type": "command",
            "command": "bash .claude/hooks/swift-post-write.sh"
          }
        ]
      }
    ]
  }
}
```

```bash
#!/bin/bash
# .claude/hooks/swift-post-write.sh

FILE_PATH="${CLAUDE_TOOL_OUTPUT_FILE_PATH}"

if [[ "$FILE_PATH" == *.swift ]]; then
  swiftformat "$FILE_PATH" --quiet 2>/dev/null
fi
```

SwiftFormat은 수정이 필요없으면 파일을 그대로 둔다. 부수효과 없음.

---

## 실제 효과

```
훅 없을 때: AI 생성 → lint 실패 → 개발자 수동 확인 → 재지시 → 재생성 (평균 3회)
훅 있을 때: AI 생성 시도 → 훅 차단 → AI 자동 수정 → 통과 (1회)
```

**주의사항**:
- SwiftLint, SwiftFormat이 로컬에 설치되어 있어야 함 (`brew install swiftlint swiftformat`)
- CI 환경에도 동일 버전 설치 필요 (버전 불일치 시 CI만 실패하는 상황 발생)
- `error` 레벨만 차단 권장. `warning`까지 차단하면 AI가 무한 루프에 빠질 수 있음
