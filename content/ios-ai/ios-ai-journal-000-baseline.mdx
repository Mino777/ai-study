## 한 줄 요약

RIBs + ReactorKit 기반 iOS 앱에 AI 운영 환경 베이스라인을 설치했다. Claude가 아키텍처를 이해하고, 안전하게 코드를 생성하며, 코드 규칙을 자동으로 지키는 최소 구조.

---

## 문제 상황

iOS 앱 개발에 Claude를 처음 투입했을 때의 상황:

| 문제 | 현상 |
|------|------|
| 아키텍처 무지 | RIBs 레이어 경계를 무시하고 Presentation에서 Repository 직접 호출 |
| 코드 규칙 무시 | SwiftLint 경고 유발 코드 생성, SwiftFormat 스타일 불일치 |
| 컨텍스트 오염 | `.build/` 폴더(130GB+) 접근 시도로 토큰 낭비 |
| 반복 설명 비용 | 매 세션마다 프로젝트 구조를 처음부터 설명해야 함 |

해결책은 "Claude가 기억해야 할 규칙"이 아니라 "구조가 강제하는 가드"다.

---

## 아키텍처: RIBs + ReactorKit

프로젝트 레이어 구조는 고정되어 있다. AI가 이걸 모르면 반드시 레이어 위반 코드를 생성한다.

```
Presentation (View + Reactor)
    ↕ Action / State (ReactorKit 단방향)
UseCase (비즈니스 로직)
    ↕
Repository (데이터 추상화)
    ↕
Network / Storage (외부 접근)
```

### ReactorKit 패턴

```swift
// Action → mutate() → Mutation → reduce() → State
// View는 Action만 보냄. State만 구독.
class HomeReactor: Reactor {
    enum Action {
        case viewDidLoad
        case refresh
    }

    enum Mutation {
        case setLoading(Bool)
        case setItems([Item])
    }

    struct State {
        var isLoading = false
        var items: [Item] = []
    }

    func mutate(action: Action) -> Observable<Mutation> {
        switch action {
        case .viewDidLoad, .refresh:
            return Observable.concat([
                .just(.setLoading(true)),
                homeUseCase.fetchItems().map { .setItems($0) },
                .just(.setLoading(false))
            ])
        }
    }

    func reduce(state: State, mutation: Mutation) -> State {
        var state = state
        switch mutation {
        case .setLoading(let v): state.isLoading = v
        case .setItems(let v): state.items = v
        }
        return state
    }
}
```

**핵심 규칙**:
- View → Reactor: Action만
- Reactor → View: State만
- Reactor는 UseCase만 호출, Repository는 직접 접근 금지
- 레이어 간 직접 접근 = 코드 리뷰 거부 사유

## 베이스라인 설치 항목

### 1. CLAUDE.md — 프로젝트 계약서

모든 세션이 시작할 때 Claude가 첫 번째로 읽는 파일. 여기에 없으면 Claude는 모른다고 가정한다.

포함 내용:
- RIBs 레이어 구조 + 금지 패턴 명시
- SwiftLint / SwiftFormat 규칙 요약
- 커스텀 스킬 5종 목록 및 용도
- 절대 건드리지 말아야 할 파일/폴더 목록
- Persistent 모듈(AES256 암호화 UserDefaults) 사용 규칙

### 2. .claudeignore — 컨텍스트 오염 차단

```
.build/
.swiftpm/
Pods/
DerivedData/
*.xcuserstate
*.xcworkspace/xcuserdata/
Generated/
R.generated.swift
TestResults/
*.xcresult
```

`.build/` 하나만으로 130GB+ 절감. 이게 없으면 Claude는 빌드 아티팩트를 소스 코드로 착각한다. [자세한 내용 → Ep.003](/wiki/ios-ai/ios-ai-journal-003-claudeignore-spm)

### 3. 커스텀 스킬 5종

`.claude/commands/` 에 배치. 반복적인 작업 흐름을 구조화한다.

| 스킬 | 용도 |
|------|------|
| `/ios-triage` | 버그 접수 → 인터뷰식 원인 파악 → 브리프 작성 |
| `/ios-review` | PR 리뷰 (RIBs 준수 + 메모리 관리 + 코드 품질) |
| `/ios-investigate` | 크래시/에러 심층 분석 (스택 트레이스 해석) |
| `/ios-ship` | 앱스토어 배포 플로우 (빌드 → 심사 → 배포) |
| `/ios-arch` | 아키텍처 설계 상담 (새 RIB 설계, 모듈 분리) |
