## iOS AI Journal 000: 온디바이스 동적 컨텍스트 주입 기초

iOS 앱에서 AI를 활용할 때 가장 핵심은 **올바른 시점에 올바른 컨텍스트를 제공하는 것**이다.

### 왜 동적 컨텍스트가 필요한가?

정적 프롬프트는 앱의 현재 상태를 알 수 없다:
- 사용자가 지금 보고 있는 화면
- 현재 활성화된 세션의 정보
- 최근 사용자 인터랙션 히스토리
- 디바이스의 환경 정보 (위치, 네트워크, 배터리)

동적 컨텍스트 주입이 해결하는 문제:
1. **관련성(Relevance)**: 사용자의 현재 상황에 맞는 응답
2. **정확성(Accuracy)**: 최신 데이터를 기반으로 한 판단
3. **반응성(Responsiveness)**: 실시간으로 변화하는 환경에 대응

### iOS에서 컨텍스트 수집의 3가지 레이어

```
┌─────────────────────────────────────┐
│  Layer 1: 앱 내부 상태               │
│  (View State, User Input, Session)  │
├─────────────────────────────────────┤
│  Layer 2: iOS System Context        │
│  (Location, Contacts, Calendar)     │
├─────────────────────────────────────┤
│  Layer 3: 네트워크/외부 데이터       │
│  (API 응답, 실시간 정보)             │
└─────────────────────────────────────┘
```

이번 시리즈에서는 이 3가지 레이어를 각각 어떻게 안전하고 효율적으로 수집하고, AI에 주입할 것인지를 다룬다.

### Baseline: 가장 단순한 형태

```swift
// 가장 기본적인 동적 컨텍스트 주입
struct AIContext {
    let currentScreenName: String
    let userSessionId: String
    let recentActions: [String]
    let timestamp: Date
}

func askAI(query: String, context: AIContext) {
    let prompt = """
    사용자가 현재 \(context.currentScreenName) 화면에서 
    \(query)를 요청했습니다.
    
    최근 액션: \(context.recentActions.joined(separator: ", "))
    시간: \(context.timestamp)
    """
    // API 호출
}
```
