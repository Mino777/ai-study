## 한 줄 요약

SPM 멀티모듈 iOS에서 `.claudeignore` 없이 Claude를 쓰면 **컨텍스트 130GB 폭발**. `.build/` 하나만 ignore하면 99% 놓치고, **`**/.build/`** 글로브 패턴이 필수다.

---

## 사고 경위

Claude Code에게 프로젝트 구조를 파악해달라고 했을 때 일어난 일.

```
Claude: (Glob 실행 중)
Claude: (Read 실행 중)
Claude: 컨텍스트 창 초과. 응답 불가.
```

`.build/` 용량을 재보자.

```bash
$ du -sh .build/
6.8G    .build/
```

"엥, 7GB면 버틸 만한데?" — 바로 그 지점이 함정이었다. 루트 `.build/`만 본 거다.

```bash
$ du -sh */.build/ 2>/dev/null
6.8G    Bridge/.build/
6.5G    Common/.build/
6.7G    Account/.build/
6.9G    Home/.build/
...
$ du -ch **/.build/ | tail -1
130G    total
```

각 SPM 모듈마다 **자기 자신의 `.build/`를 들고 있다.** 합쳐서 130GB.

---

## 왜 SPM 멀티모듈에서 `.build/`가 130GB인가

SPM 멀티모듈 프로젝트는 각 모듈이 **독립 `Package.swift`** 를 가진다. 빌드 시스템 관점에서 각 모듈은 자기 자신의 빌드 캐시를 가져야 한다.

```
repo/
├── Bridge/
│   ├── Package.swift
│   └── .build/          ← 6~7GB
├── Common/
│   ├── Package.swift
│   └── .build/          ← 6~7GB
├── Account/
│   ├── Package.swift
│   └── .build/          ← 6~7GB
├── Home/
│   ├── Package.swift
│   └── .build/          ← 6~7GB
... (20+ 모듈)
```

`.build/` 하나당 debug + release + `arm64-apple-ios-simulator` + `arm64-apple-ios` 4벌의 `.o` / `.swiftmodule` / `.swiftdoc`가 들어간다. 모듈당 6~7GB는 과장이 아니다.

**20+ 모듈 × 6~7GB = ~130GB.**

---

## 결정적 함정: 루트 `.build/`만으로는 99% 놓친다

처음에 순진하게 이렇게 썼다.

```gitignore
# .claudeignore (잘못된 버전)
.build/
```

Claude가 Glob을 돌리면 여전히 `Bridge/.build/`, `Common/.build/`, `Account/.build/` 를 전부 들어간다. `.build/` 라는 패턴은 **루트 디렉토리 한 개만** 차단한다.

정답은 **글로브 패턴**이다.

```gitignore
.build/
**/.build/
```

`**/.build/`는 "어느 깊이에 있든 `.build/` 디렉토리는 모두 제외" 라는 뜻이다. 이 한 줄이 130GB 중 129.9GB를 차단한다.

## moneyflow-ios `.claudeignore` 전체

실전에서 돌고 있는 전체 파일이다.

```gitignore
# SPM build artifacts — 각 모듈 하위에 .build/ 존재, 모듈당 6-7GB
# Bridge/.build, Common/.build, Account/.build ... 전부 제외
.build/
**/.build/

# CocoaPods (63MB)
vendor/

# SPM package sources (3.7MB)
external-spm/

# Xcode project XML — Claude가 직접 편집하지 않는 generated 파일
MoneyFlow.xcodeproj/

# Lock files
Gemfile.lock

# Ruby bundle
.bundle/

# gstack internal
.gstack/
```

핵심 항목은 네 줄이다.

- `**/.build/` — SPM 멀티모듈 `.build/` 전부 (130GB)
- `vendor/` — CocoaPods 벤더 디렉토리 (63MB, AI가 편집 불필요)
- `external-spm/` — 외부 SPM 패키지 소스 미러 디렉토리 (수 MB, AI가 편집 불필요)
- `*.xcodeproj/` (여기선 `MoneyFlow.xcodeproj/` 등 프로젝트별 이름) — Xcode generated XML, AI가 직접 편집하지 않음

## 적용 전후

| | 적용 전 | 적용 후 |
|---|---|---|
| 프로젝트 색인 크기 | ~130GB | ~수십 MB |
| Claude 응답 | 컨텍스트 초과로 불가 | 정상 |
| Glob/Read 토큰 소비 | 측정 불가 (즉시 터짐) | 정상 수준 |
