## 한 줄 요약

SPM 멀티모듈 iOS 프로젝트에서 `.claudeignore` 없이 Claude를 쓰면 컨텍스트 창이 즉각 초과된다. `.build/` 하나가 130GB. 실제 겪었고, `.claudeignore` 하나로 해결됐다.

---

## 사고 경위

Claude Code에게 프로젝트 구조를 파악해달라고 했을 때 일어난 일:

```
Claude: (Glob 실행 중)
Claude: (Read 실행 중)
Claude: 컨텍스트 창 초과. 응답 불가.
```

원인 분석:

```bash
$ du -sh .build/
130G    .build/
```

---

## 왜 .build/ 가 130GB인가

SPM(Swift Package Manager)은 빌드 아티팩트를 `.build/` 에 저장한다. 멀티모듈 프로젝트에서는:

```
.build/
├── debug/
│   ├── ModuleA.o        (각 소스 파일의 Object 파일)
│   ├── ModuleA.swiftdoc
│   ├── ModuleA.swiftmodule
│   └── ... (모듈 수 × 파일 수)
├── release/
│   └── ... (debug와 동일한 구조)
├── arm64-apple-ios-simulator/   (시뮬레이터 빌드)
│   └── ...
└── arm64-apple-ios/             (디바이스 빌드)
    └── ...
```

모듈 10개 × 소스 파일 100개 × 4벌 = 4000개 파일. Debug + Release 각각 바이너리 포함.

---

## Claude가 이걸 읽으면?

Glob이나 Read로 프로젝트를 파악할 때 `.build/` 안의 수천 개 파일도 포함된다. 이것들은 전부 컴파일된 바이너리와 임시 파일이라 읽어도 아무 의미가 없다.

결과:
1. **컨텍스트 창 초과** → 응답 불가
2. **토큰 낭비** → 비용 폭증
3. **잘못된 분석** → AI가 소스 코드 대신 빌드 아티팩트를 "소스"로 착각

---

## 해결: .claudeignore 전체 구성

```gitignore
# .claudeignore

# === SPM 빌드 아티팩트 (최우선 제외) ===
.build/
.swiftpm/
*.o
*.d
*.swiftmodule
*.swiftdoc

# === CocoaPods ===
Pods/
Podfile.lock

# === Xcode 빌드 아티팩트 ===
DerivedData/
*.xcuserstate
*.xcworkspace/xcuserdata/
*.xcodeproj/xcuserdata/

# === 자동 생성 파일 ===
R.generated.swift
Generated/
*.generated.swift

# === 테스트 결과 ===
TestResults/
*.xcresult
fastlane/report.xml

# === 대용량 바이너리 ===
*.ipa
*.dSYM/
*.app
```

## 적용 전후 비교

| | 적용 전 | 적용 후 |
|---|---------|---------|
| 프로젝트 색인 크기 | ~130GB (컨텍스트 초과) | ~수십 MB |
| Claude 응답 | 컨텍스트 초과로 불가 | 정상 |
| 토큰 소비 | 측정 불가 | 정상 수준 |

## SPM 멀티모듈: Claude가 놓치는 부분

`.claudeignore`로 컨텍스트 오염은 막았지만, SPM 멀티모듈 특유의 함정이 남는다.

**AI가 새 파일을 추가할 때 자동으로 하지 못하는 것들**:

```markdown
## CLAUDE.md에 명시

### 모듈에 파일 추가 시 사람이 직접 해야 할 일
1. Package.swift에 타겟 소스 파일 목록 업데이트 (필요한 경우)
2. Xcode 타겟 멤버십 확인 (AI가 가장 많이 놓치는 부분)
3. 모듈 간 의존성 추가 시 Package.swift 수정 + Xcode 재빌드

AI가 Swift 파일을 생성했다고 해서 빌드 타겟에 자동으로 포함되지 않는다.
반드시 Xcode에서 파일 멤버십을 확인할 것.
```

## 핵심

`.claudeignore`는 웹 프로젝트의 `.gitignore`와 같다. 없으면 작동하지 않는다. iOS 프로젝트를 Claude와 함께 쓰려면 **Day 1에 설치**하는 것이 맞다.

[베이스라인 Ep.000](/wiki/ios-ai/ios-ai-journal-000-baseline)에 포함된 이유가 이것이다.
