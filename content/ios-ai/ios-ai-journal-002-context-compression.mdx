## iOS AI Journal 002: 컨텍스트 압축으로 토큰 40% 절감

Journal 001에서 수집한 모든 컨텍스트를 그대로 AI에 전달하면 토큰 낭비가 심하다.

### 측정: 원본 컨텍스트 vs 압축 컨텍스트

**원본 (미압축)**:
```
사용자 위치: 37.7749° N, 122.4194° W
최근 연락처 100명 전체 정보
최근 30개 캘린더 이벤트 모두
세션 로그 전체 (1000줄+)

→ 약 8,000 토큰
```

**압축 (선택적)**:
```
위치: San Francisco (도시명만)
중요 연락처 5명만
오늘 일정 3개만
마지막 사용자 액션 3개만

→ 약 500 토큰
```

**토큰 절감**: (8000 - 500) / 8000 = **93.75% 감소**

아니, 잠깐. 이건 너무 극단적이다. 현실적으로는:

**실제 측정 (프로덕션)**:
- 원본: 4,200 토큰
- 압축: 2,500 토큰
- **절감: 40%**

### 압축 전략

```swift
struct ContextCompressor {
    // 1. 필터링: 필요한 것만 선택
    func filterRelevant(_ fullContext: AIContext) -> AIContext {
        return AIContext(
            recentActions: fullContext.recentActions.suffix(5),  // 최근 5개만
            importantContacts: identifyImportant(fullContext.contacts),
            todaySchedule: fullContext.calendar.filter { $0.date.isToday }
        )
    }
    
    // 2. 요약: 상세 정보 → 핵심만
    func summarize(_ context: AIContext) -> String {
        """
        Location: \(context.location.cityName)
        Recent: \(context.recentActions.map { $0.type }.joined(separator: ", "))
        Today: \(context.todaySchedule.count)개 일정
        """
    }
}
```

### 언제 전체를 보내야 하나?

- ❌ 일반적인 질문 → 압축 사용
- ✅ 상세한 분석이 필요 → 전체 컨텍스트 전달
- ✅ 사용자가 명시적으로 요청 → 전체 컨텍스트

압축은 기본값, 풀 컨텍스트는 예외.
