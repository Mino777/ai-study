## iOS AI Journal 004: 응답 속도 vs 정확성 트레이드오프

Journal 003까지의 방법들은 각각 장단점을 가진다.

### 3가지 전략의 비교

| 전략 | 응답 속도 | 정확성 | 토큰 비용 |
|------|---------|--------|---------|
| 스냅샷 | ⚡⚡⚡ (500ms) | ⭐⭐ | 💰 (낮음) |
| Function Calling | ⚡⚡ (2000ms) | ⭐⭐⭐⭐⭐ | 💰💰 (높음) |
| 병렬 + 캐싱 | ⚡⚡⚡ (600ms) | ⭐⭐⭐ | 💰💰 (중간) |

### 실전: 상황별 최적 전략

**빠른 응답이 중요한 경우**
- 검색, 자동완성, 실시간 제안
- → **스냅샷 + 캐싱** 사용

**정확성이 중요한 경우**
- 금융 거래, 의료 정보, 약속 확인
- → **Function Calling** 사용 (느려도 정확함)

**균형**
- 대부분의 채팅, 추천
- → **병렬 수집 + 선택적 Function Calling**

### 병렬 + 캐싱 패턴

```swift
struct SmartContextManager {
    var cache: [String: Any] = [:]
    
    async func getContext(priority: Priority) -> AIContext {
        switch priority {
        case .fast:
            // 캐시된 데이터 사용 (500ms 이내)
            return getCachedContext()
            
        case .accurate:
            // 최신 데이터 조회 (2000ms)
            let fresh = await updateAllData()
            cache.updateValues(fresh)
            return fresh
            
        case .balanced:
            // 병렬: 캐시는 즉시, 백그라운드에서 업데이트
            let cached = getCachedContext()
            Task { cache = await updateAllData() }
            return cached
        }
    }
}
```

### 사용자 심리

- 500ms 이상 지연 → "앱이 느리다"
- 2초 지연 → "뭔가 생각하고 있다" (허용됨)
- 3초 이상 → 사용자가 포기

따라서 **응답 버튼은 빠르게, 내용 정확성은 뒤따라감**.
