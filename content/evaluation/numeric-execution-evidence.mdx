## 갭 / 맥락

"Tests passed ✅"를 그대로 신뢰하면 안 되는 이유를 실전에서 겪었다. Aidy Session 4에서 iOS 워커가 R2부터 R9까지 8라운드 동안 매 라운드 "테스트 통과"로 보고했고, Gate는 모두 PASS로 처리했다.

QA 단계에서 실제 xcodebuild 출력을 확인하자:

```
Test Suite 'All tests' started at ...
Test Suite 'All tests' passed at ...
Executed 0 tests, with 0 failures (0 unexpected) in 0.000 (0.001) seconds
** TEST SUCCEEDED **
```

**0 tests executed.** 8라운드 동안 iOS 테스트는 한 번도 실행되지 않았다. `exit 0`이 반환되어 "통과"로 해석됐을 뿐.

---

## 핵심 원칙

### "통과"는 두 개의 클레임을 섞고 있다

```
"Tests passed" = [실행됐다] + [실패가 없었다]
```

무실행 상태에서 exit 0이 반환되면 첫 번째 클레임(실행)이 거짓인데도 두 번째 클레임(실패 없음)만 보고 "통과"로 해석된다.

### Numeric Execution Evidence 요구

워커/CI가 테스트 결과를 보고할 때 **반드시** 다음 형태여야 함:

```
Test run with {NN} tests, {K} failures
```

- `NN` (카디널리티): 실제 실행된 테스트 수. **NN = 0이면 FAIL**
- `K` (실패 수): **K > 0이면 FAIL**

자동 체크:

```bash
# Gate 1 스크립트 예시
if ! echo "$output" | grep -E "run with [1-9][0-9]* tests"; then
  echo "GATE 1 FAIL: 테스트 실행 증거(숫자) 없음 또는 0개"
  exit 1
fi
```

### 플랫폼별 숫자 증거 패턴

| 플랫폼 | 도구 | 숫자 증거 패턴 |
|--------|------|---------------|
| Swift | xcodebuild | `Executed {NN} tests, with {K} failures` |
| Kotlin (JVM) | Gradle | `{NN} tests completed, {K} failed` |
| JS/TS | vitest | `Test Files NN passed / Tests NN passed` |
| Python | pytest | `collected NN items` + `NN passed` |

패턴이 없거나 NN = 0이면 Gate FAIL.

## Silent Failure — 가장 위험한 실패 유형

| 유형 | 증상 | 발견 난이도 |
|------|------|-----------|
| **Loud failure** | 에러 로그 + non-zero exit | 낮음 |
| **Semi-silent** | 에러 로그 + exit 0 | 중간 |
| **Silent (no-op)** | "no tests to run" + exit 0 | **높음 — 가장 위험** |

Silent failure는 성공 신호와 구분이 안 되므로, **명시적 긍정 증거(숫자)를 요구하는 방식**으로만 막을 수 있다. "없음을 확인하기"가 아니라 "있음을 확인하기".

## AI Agent Directive

### Trigger
- 워커 Claude 세션이 "테스트 통과"를 보고하는 모든 시점
- Gate 1/Gate 2 체크리스트 설계 시
- CI/CD 파이프라인의 테스트 스텝 설계 시

### Actionable Steps
1. **Gate 체크리스트에 명시**: "테스트 실행 증거(NN tests run) 출력 없으면 FAIL"
2. **워커 프롬프트 템플릿 수정**: 테스트 완료 보고 시 숫자 포함 의무화
3. **자동 파싱 스크립트**: 플랫폼별 숫자 증거 패턴 regex로 추출
4. **NN = 0도 FAIL 처리**: "실행할 테스트 없음"은 구현 부재의 증거

### Anti-patterns
- **"Tests passed" 메시지만으로 PASS** — 숫자 없으면 FAIL
- **exit code만으로 판단** — exit 0은 통과와 무실행을 동시에 포함
- **자연어 보고 신뢰** — 워커 AI가 "모두 통과"라고 해도 원본 출력 확인 필수
