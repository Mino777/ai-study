## 갭 / 맥락

moneyflow는 투자 분석 콘텐츠를 블로그 → 유튜브 쇼츠 → 인스타그램 릴스 → 뉴스레터로 멀티플랫폼 발행한다. 문제는 플랫폼 간 **의존성**:

- 인스타 릴스는 유튜브 쇼츠를 기반으로 만듦
- 유튜브 쇼츠는 유튜브 스크립트가 필요
- 유튜브 스크립트는 블로그 원문에서 파생
- 뉴스레터는 블로그 요약

수동으로 순서를 관리하면 실수가 발생한다. DAG로 의존성을 선언하고 실행 순서를 자동 계산하면 해결.

### 의존성 그래프 선언

```typescript
// 8가지 콘텐츠 타입의 의존성
const DEPENDENCIES: Record<ContentType, ContentType[]> = {
  blog:            [],                    // 원본 — 의존성 없음
  youtube_script:  ["blog"],              // 블로그에서 파생
  youtube_shorts:  ["youtube_script"],    // 스크립트에서 파생
  ig_reel:         ["youtube_shorts"],    // 쇼츠를 릴스로 변환
  ig_carousel:     ["blog"],             // 블로그 → 카드 슬라이드
  newsletter:      ["blog"],             // 블로그 요약
  thread:          ["blog"],             // X/Twitter 스레드
  podcast_script:  ["blog"],             // 블로그 → 팟캐스트 대본
};
```

### 실행 순서 자동 계산

```typescript
function getExecutionOrder(requested: ContentType[]): ContentType[] {
  const result: ContentType[] = [];
  const visited = new Set<ContentType>();

  function resolve(type: ContentType) {
    if (visited.has(type)) return;
    visited.add(type);

    // 전이적 의존성을 먼저 해소
    for (const dep of DEPENDENCIES[type]) {
      resolve(dep);
    }
    result.push(type);
  }

  for (const type of requested) {
    resolve(type);
  }

  return result; // 중복 제거된 실행 순서
}

// 예: getExecutionOrder(["ig_reel"])
// → ["blog", "youtube_script", "youtube_shorts", "ig_reel"]
//   (4개가 자동으로 추가됨!)
```

### 순환 의존성 감지

```typescript
function detectCircular(deps: Record<string, string[]>): string[] | null {
  const visiting = new Set<string>();
  const visited = new Set<string>();

  function dfs(node: string, path: string[]): string[] | null {
    if (visiting.has(node)) return [...path, node]; // 순환 발견!
    if (visited.has(node)) return null;

    visiting.add(node);
    for (const dep of deps[node] ?? []) {
      const cycle = dfs(dep, [...path, node]);
      if (cycle) return cycle;
    }
    visiting.delete(node);
    visited.add(node);
    return null;
  }

  for (const node of Object.keys(deps)) {
    const cycle = dfs(node, []);
    if (cycle) return cycle;
  }
  return null;
}
```

## AI Agent Directive

### Trigger
- 멀티플랫폼 콘텐츠 발행 파이프라인을 설계할 때
- 작업 간 의존성이 있는 자동화 워크플로를 구축할 때

### Actionable Steps
1. **의존성 맵 선언** — 각 작업 타입의 선행 조건을 명시적으로 정의
2. **getExecutionOrder()** — 전이적 의존성을 재귀적으로 해소 + 중복 제거
3. **순환 감지** — DFS 기반 cycle detection을 빌드/테스트 타임에 실행
4. **스케줄 계산** — 플랫폼별 발행 시차를 분 단위로 관리

### Anti-patterns
- **의존성을 암묵적으로 관리** — "블로그 먼저 올려야 해"를 코드가 아니라 사람 기억에 의존
- **순환 감지 생략** — 프로덕션에서 무한 루프 발생 시 비용 폭주
